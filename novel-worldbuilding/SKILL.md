# 小说世界观管理工具

**核心理念**：世界观是小说的"宪法"，所有创作工具必须在世界观框架内运行，不能自相矛盾。

---

## 使用方式

### 创建世界观
- `/novel-worldbuilding 新建 [小说名]` — 从零创建世界观设定文档
- `/novel-worldbuilding 从灵感 [灵感文件路径]` — 从已有灵感提取世界观
- `/novel-worldbuilding 从大纲 [大纲文件路径]` — 从已有大纲提取世界观

### 管理世界观
- `/novel-worldbuilding 查看 [小说名]` — 查看当前世界观设定
- `/novel-worldbuilding 编辑 [小说名]` — 编辑世界观设定
- `/novel-worldbuilding 检查 [小说名] [待检查文件]` — 检查文件是否符合世界观

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：`world_[小说名].md`

## 一致性检查规则

### 检查维度

| 维度 | 检查内容 | 违反后果 |
|------|----------|----------|
| **等级体系** | 角色等级是否符合设定 | 硬伤 |
| **势力关系** | 势力间关系是否前后一致 | 硬伤 |
| **地理距离** | 地点间距离/方位是否矛盾 | 硬伤 |
| **时间线** | 事件时间顺序是否合理 | 硬伤 |
| **能力限制** | 角色能力是否超出设定限制 | 中等问题 |
| **规则违反** | 是否违反已建立的世界规则 | 硬伤 |
| **物品追踪** | 关键物品的去向是否明确 | 小问题 |
| **角色状态** | 角色当前状态是否正确 | 中等问题 |

### 检查流程

```
1. 读取世界观文档
2. 读取待检查文件
3. 逐项检查：
   - 提取文件中的世界观相关名词（地名、势力名、等级名、能力名）
   - 与世界观文档对比
   - 检查是否矛盾
4. 输出检查报告
```

### 检查报告格式

```markdown
# 世界观一致性检查报告

## 检查文件：[文件路径]
## 检查时间：[时间]

### 硬伤（必须修复）
1. [问题描述]：[具体矛盾] → 建议修改为：[正确内容]

### 中等问题（建议修复）
1. [问题描述]：[具体问题] → 建议：[修改方向]

### 小问题（可选修复）
1. [问题描述]：[具体问题]

### 检查通过
- [维度1]：✓ 一致
- [维度2]：✓ 一致
...
```

## 与其他工具的协作

### 灵感阶段（novel-inspiration）
- 灵感生成时，自动创建世界观初稿
- 包含：世界类型、空间结构、力量体系的基础框架

### 大纲阶段（novel-outline）
- 大纲生成时，扩展世界观：
  - 添加势力分布
  - 添加地理环境
  - 添加历史事件框架
  - 添加等级体系详细设计

### 章节大纲阶段（novel-chapter-outline）
- 生成章节大纲时，更新世界观：
  - 标注每个势力/地点首次出现的章节
  - 添加伏笔追踪表

### 正文阶段（novel-chapter）
- 生成正文前，必须读取世界观文档
- 生成正文后，自动更新世界观：
  - 更新角色当前状态
  - 更新事件记录
  - 更新伏笔状态
  - 更新物品追踪

---

## 输出文档规则

### 创建条件
- 新建小说时必须创建世界观文档
- 灵感阶段自动生成初稿
- 大纲阶段自动扩展

### 更新时机
- 每完成一章正文后自动更新
- 用户手动编辑后需重新检查一致性

### 文件管理
- 每部小说一个世界观文档
- 文件名：`world_[小说名].md`
- 与其他创作文件保存在同一目录
