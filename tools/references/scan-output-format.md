# 扫榜输出格式规范

## 数据采集格式

每本书的采集数据结构：

```
- 书名：《XXX》
- 作者：XXX
- 平台：起点/番茄/晋江/七猫/刺猬猫
- 题材：玄幻修仙/都市/历史/...
- 字数：XX万字
- 状态：连载/完结
- 排名：榜单名+排名
- 标签：[标签1, 标签2, ...]
- 开篇卖点：（前3句话提取）
- 核心设定：（一句话提炼）
```

## 趋势分析格式

```
## [平台名] 趋势分析

### 数据概览
- 采集时间：YYYY-MM-DD
- 样本量：XX 本
- 覆盖榜单：XXX

### 题材分布
| 题材 | 数量 | 占比 | 趋势 |
|------|------|------|------|
| XXX | XX | XX% | ↑/↓/→ |

### 高频关键词
1. XXX（出现 XX 次）
2. XXX（出现 XX 次）
...

### 核心发现
- 发现1：...
- 发现2：...
```

## 选题决策格式

```
## 选题决策报告

### 推荐方向
1. [方向名] — 热度★★★★☆，竞争度中，可行性高
2. [方向名] — 热度★★★☆☆，竞争度低，可行性中

### 详细分析
[每个方向的详细分析]

### 最终建议
[综合建议]
```
