# 小说正文生成系统（单章 + 批量双模式）

**核心理念**：正文不是一次生成的，而是通过"生成→评分→优化"的迭代循环打磨出来的。单章模式追求极致（9.0+），批量模式追求高效稳定（8.5+）。

**双模式设计**：一个skill同时支持单章精写和批量生产，共享6阶段流水线，差异化质量阈值和流程控制。

---

## 使用方式

### 单章模式（默认）

```
/novel-chapter 第X章                              # 按章节号生成（自动查找对应大纲）
/novel-chapter [章节大纲文件路径]                    # 根据章节大纲生成正文
/novel-chapter [章节大纲文件路径] --previous [上一章]  # 指定上一章内容
```

#### 单章模式参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--focus 亲密/幽默/日常/惊喜` | 本章重点强化某类元素 | 无 |
| `--words 5000` | 自定义目标字数 | 4000 |
| `--max 3` | 最多迭代轮数 | 5 |
| `--target 8.5` | 自定义目标分数 | 9.0 |

#### 单章达标标准

- **默认目标分数：9.0**（极品正文）
- **达标判定：总分 >= 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

---

### 批量模式

```
/novel-chapter --batch                              # 交互式选择批量生成范围
/novel-chapter 批量                                  # 同上（中文触发）
/novel-chapter --batch --from 第X章                  # 从指定章节开始批量生成
/novel-chapter --batch --to 第Y章                    # 生成到指定章节
/novel-chapter --batch --from 第X章 --to 第Y章        # 指定范围
/novel-chapter --batch --resume                      # 从上次中断处继续
/novel-chapter --batch --status                      # 查看当前批量进度
```

#### 批量模式参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--batch-size 10` | 每批生成章节数 | 10 |
| `--target 8.0` | 自定义目标分数 | 8.5 |
| `--resume` | 断点续写 | — |
| `--status` | 查看进度 | — |
| `--from 第X章` | 起始章节 | 第1章 |
| `--to 第Y章` | 结束章节 | 全部 |

#### 批量达标标准

- **默认目标分数：8.5**（批量模式阈值，低于单章的9.0）
- **达标判定：单章总分 >= 8.5** 才算完成
- **未达标处理**：自动优化一轮，仍不达标则标记"需手动优化"并继续下一章
- **批次通过条件**：该批次平均分 >= 8.0 才进入下一批

## 完整流程概览（6阶段）

```
┌─────────────────────────────────────────────────────────────┐
│  阶段一：场景构思（brainstorming）                            │
│  头脑风暴 → 探索场景表现方式 → 确定写作重点                   │
├─────────────────────────────────────────────────────────────┤
│  阶段二：资料支撑（novel-manage + WebSearch）                  │
│  读取世界观 → 查询专业细节 → 记录关键信息 → 增加真实感        │
├─────────────────────────────────────────────────────────────┤
│  阶段三：正文生成（novel-writing）                            │
│  开篇承接 → 场景1 → 场景2 → 场景3 → 收尾钩子 → 初始正文      │
├─────────────────────────────────────────────────────────────┤
│  阶段四：体验评分（novel-quality 量表C）                      │
│  三层层级评价 → 识别体验短板 → 判断是否达标                   │
├─────────────────────────────────────────────────────────────┤
│  阶段五：吸引力优化 + 去AI味                                  │
│  针对短板优化 → 去AI味（novel-deslop）→ 补充钩子 → 再评分     │
├─────────────────────────────────────────────────────────────┤
│  阶段六：深度分析 + 世界观更新                                │
│  读者体验分析 → 更新novel-manage → 输出最终报告               │
└─────────────────────────────────────────────────────────────┘
```

> **批量模式**：上述6阶段对每一章独立执行，外加批次级的一致性检查和进度管理。

## 阶段一：场景构思（brainstorming）

**工具**：内部头脑风暴

**目标**：在正式生成正文前，先进行头脑风暴，探索场景的最佳表现方式。

### 执行流程

```
1. 读取章节大纲，理解本章的核心任务
2. 读取 novel-manage 中的世界观设定和前文状态
3. 根据章节类型（开篇/亲密/日常/高潮/转折），进行发散思考
4. 生成3-5个不同的场景表现方向，每个方向包含：
   - 场景切入点（从哪里开始写）
   - 情绪基调（本章的主要情绪）
   - 吸引力元素植入点（亲密/幽默/悬念放在哪里）
   - 潜在风险（可能的问题）
5. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 场景构思头脑风暴

## 章节：第XX章 [章节标题]

## 章节类型：[开篇章/亲密章/日常章/高潮章/转折章]

## 核心任务：[本章要完成什么]

## 前文状态：[接续novel-manage中的世界观状态]

## 场景表现方向

### 方向A：[名称]
- 场景切入点：[从哪里开始]
- 情绪基调：[主要情绪]
- 吸引力植入：[亲密/幽默/悬念位置]
- 潜在风险：[可能的问题]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```

### 批量模式差异

- 批量模式下自动选择最优方向，不等待用户确认
- 每章自动读取 novel-manage 中的最新状态，确保与前文衔接

## 阶段二：资料支撑（novel-manage + WebSearch）

**工具**：novel-manage（世界观管理）+ WebSearch（网络搜索）

**目标**：读取世界观设定，查询相关专业细节，为正文增加真实感和代入感。

### 执行流程

```
1. 从 novel-manage 读取世界观设定（世界规则、角色档案、地点设定等）
2. 从 novel-manage 读取前文状态（角色关系、伏笔、事件记录）
3. 读取 追踪/写作反馈.md，获取管理检查的反馈约束：
   - 高频问题清单（必须避免的写作错误）
   - 角色注意事项（已知问题和写作建议）
   - 伏笔预警（即将超期/已超期的伏笔）
   - 世界观约束提醒（易错点）
   - 上一章衔接要点（结束状态、待处理悬念）
4. 根据章节内容，确定需要查询的资料类型
5. 使用 WebSearch 搜索需要补充的专业细节
6. 确保正文内容符合世界观设定和反馈约束
7. 记录本章需要特别注意的设定约束和反馈要点
```

## 阶段三：正文生成（novel-writing 去耦合化组合）

**工具**：novel-writing（创作技法工具箱 - 模块C + 去耦合分类库）

**目标**：基于场景构思和资料支撑，通过去耦合化组合生成约4000字的充满惊喜的高质量正文。

### 4000字结构

| 部分 | 字数 | 功能 | 说明 |
|------|------|------|------|
| 开篇承接 | 300-500字 | 接续前文 | 时间/地点/状态过渡 |
| 场景1 | 800-1200字 | 主要内容 | 剧情推进/角色互动 |
| 场景2 | 800-1200字 | 主要内容 | 亲密/幽默/冲突 |
| 场景3 | 800-1200字 | 主要内容 | 高潮/转折/日常 |
| 收尾钩子 | 300-500字 | 章末悬念 | 驱动追更 |

### 去耦合化组合使用（核心）

**每个场景都使用去耦合化组合，确保角色表现和情节发展充满惊喜**：

1. **角色表现组合**（从 character-traits.md 选取）：
   - 为本章涉及的角色选取：个性类型×情绪状态×行为动作×语言风格
   - 同一角色在不同场景可以有不同组合（如：高冷型×害羞vs高冷型×愤怒）
   - 确保角色之间组合不重复

2. **情节元素组合**（从 plot-elements.md 选取）：
   - 为本章选取：事件类型×事件风格×场景氛围
   - 围绕章节核心体验，选取能将其推到极致的组合
   - 结合 extensions/ 中的扩展元素（如有）

3. **组合公式**：
   ```
   角色表现 = 个性类型 × 情绪状态 × 行为动作 × 语言风格
   章节内容 = 事件类型 × 事件风格 × 场景氛围 × 角色特点
   ```

4. **示例**：
   - 场景1：高冷型×害羞情绪×咬唇行为×惜字如金语言 + 日常暧昧×心动甜蜜风格×暧昧私密氛围
   - 场景2：活泼型×愤怒情绪×跺脚行为×毒舌语言 + 误会冲突×搞笑风格×日常烟火氛围
   - 同一对角色，不同场景组合不同，表现就不同，避免单一模式

### 写作技法（来自novel-writing模块C）

#### 开篇承接技法
- 时间过渡法：[时间标志词]，[地点/状态变化]
- 场景延续法：[延续上一章场景]，[自然过渡]
- 内心独白承接法：[角色内心想法]，[引出新场景]

#### 亲密描写技法
- 感官分层法：触觉→温度→深化→反馈→力度
- 微反应刻画法：用身体微反应展现情感
- 渐进式隐喻法：小雨→中雨→大雨→雷鸣
- 克制留白法：在最高潮时突然收住
- 环境烘托法：用场景氛围配合亲密情绪
- 感官差异化法：不同角色不同"味道"

#### 幽默描写技法
- 内心独白式幽默：严肃场景+内心吐槽
- 对比式幽默：两个极端的并置
- 破折号旁白法：用破折号引出旁白
- 逻辑包装法：用歪理为亲密行为找借口
- 误会式幽默：信息错位制造喜剧
- 旁观者呆滞法：局外人的困惑反应

#### 日常细节技法
- 感官叠加法：视觉+听觉+触觉+嗅觉+味觉
- 生活细节法：饮食/穿着/居住/习惯
- 习惯标签法：给角色设计标志性习惯

#### 悬念钩子技法
- 悬念型："这时，XX发生了..."
- 转折型："就在一切顺利的时候，意外发生了。"
- 信息揭露型："他终于知道了那个秘密。"
- 对话截断型："她只说了一个字：'好。'"
- 悬念提问型："如果她知道了一切，会怎样？"

## 阶段四：体验评分（novel-quality 量表C 极端化评估）

**工具**：novel-quality（评价体系 - 量表C 极端化评估）

**目标**：量化评估正文的读者体验，识别核心体验是否做到极致。

### 评分流程

```
1. 声明本章的核心体验类型（暧昧章/悲伤章/搞笑章/诱惑章/治愈章/...）
2. 确定核心极端维度（如暧昧章→暧昧感极致度）
3. 评估三层得分
4. 检查核心极端维度是否 ≥ 9.5
5. 应用极端化修正（核心维度≥9.5则总分上浮）
6. 判断是否达标
```

### 评分框架（三层层级）

#### 第一层：章节目标达成度（40%）

根据章节类型，评价该章节是否完成了它的核心任务。

| 章节类型 | 核心任务 | 评价维度 |
|----------|----------|----------|
| 开篇章 | 让读者想看第二章 | 世界观清晰度、角色引入效果、悬念设置强度 |
| 亲密章 | 让读者心动 | 亲密描写质量、情感真实度、余韵处理 |
| 日常章 | 让读者有代入感 | 生活细节真实度、角色互动自然度、感情线推进 |
| 高潮章 | 让读者"卧槽" | 冲突张力、情感爆发力、反转惊喜度 |
| 转折章 | 让读者改变预期 | 信息揭露效果、剧情转向力度、悬念延续性 |

#### 第二层：读者体验维度（40%）

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 沉浸感 | 37.5% | 读者忘记时间，完全沉浸在故事中？ |
| 情绪波动 | 37.5% | 读者经历3种以上情绪变化？ |
| 翻页欲 | 25% | 读者迫不及待想看下一章？ |

#### 第三层：技巧执行度（20%）

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 描写质量 | 30% | 感官描写细腻，有多感官叠加？ |
| 对话质量 | 30% | 对话有潜台词，有动作穿插？ |
| 节奏控制 | 25% | 快慢交替自然，有"减压阀"？ |
| 金句记忆 | 15% | 有让人想截图/引用的句子？ |

### 评分计算

```
总分 = 第一层 x 40% + 第二层 x 40% + 第三层 x 20%

第一层得分 = (维度得分 x 维度权重) 之和
第二层得分 = 沉浸感 x 37.5% + 情绪波动 x 37.5% + 翻页欲 x 25%
第三层得分 = 描写质量 x 30% + 对话质量 x 30% + 节奏控制 x 25% + 金句记忆 x 15%
```

### 评分标准（单章模式）

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| >= 9.0 | 极品 | **采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：单章模式中，只有 >= 9.0 分的正文才会被采用并保存为最终文件。**

### 评分标准（批量模式）

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| >= 9.0 | 极品 | **采用** | 直接进入阶段六 |
| 8.5-8.9 | 优质 | **采用** | 直接进入阶段六 |
| 7.5-8.4 | 良好 | 优化一轮 | 优化后>=8.5则采用，否则标记"需手动优化" |
| 6.5-7.4 | 一般 | 优化一轮 | 优化后>=8.5则采用，否则标记"需手动优化" |
| < 6.5 | 较差 | 标记跳过 | 标记"需手动优化"，继续下一章 |

**批量模式中，>= 8.5 分的正文自动采用；优化一轮后仍 < 8.5 的标记为"需手动优化"，不阻塞批量流程。**

## 阶段五：吸引力优化 + 去AI味

**工具**：novel-deslop（去AI味系统）+ 内部优化策略

**目标**：针对评分短板进行定向优化，去除AI味，补充吸引力元素。

### 优化策略（根据短板选择）

#### 第一层优化：章节目标达成度

| 章节类型 | 短板 | 优化技法 | 来源 |
|----------|------|----------|------|
| 开篇章 | 世界观混乱 | 场景构建 | novel-writing 模块C |
| 开篇章 | 角色无辨识度 | 核心六要素 | novel-writing 模块A |
| 开篇章 | 钩子弱 | 钩子设置技巧 | novel-writing 模块B |
| 亲密章 | 描写粗糙 | 感官分层法 | novel-writing 模块C |
| 亲密章 | 情感虚假 | 微反应刻画法 | novel-writing 模块C |
| 亲密章 | 余韵差 | 克制留白法 | novel-writing 模块C |
| 日常章 | 细节干瘪 | 感官叠加法 | novel-writing 模块C |
| 日常章 | 互动生硬 | 生活细节法 | novel-writing 模块C |
| 高潮章 | 冲突弱 | 冲突设计模式 | novel-writing 模块B |
| 高潮章 | 反转差 | 信息差驱动法 | novel-writing 模块B |
| 转折章 | 揭露弱 | 表里双层叙事法 | novel-writing 模块A |

#### 第二层优化：读者体验维度

| 短板 | 优化技法 | 来源 | 优化方法 |
|------|----------|------|----------|
| 沉浸感差 | 感官叠加法 | novel-writing 模块C | 增加多感官描写 |
| 情绪平淡 | 渐进式隐喻法 | novel-writing 模块C | 增加情绪层次 |
| 翻页欲弱 | 钩子设置技巧 | novel-writing 模块B | 强化章末钩子 |

#### 第三层优化：技巧执行度

| 短板 | 优化技法 | 来源 | 优化方法 |
|------|----------|------|----------|
| 描写粗糙 | 感官分层法 | novel-writing 模块C | 细化感官层次 |
| 对话生硬 | 角色语言风格差异化 | novel-writing 模块C | 增加个性化语言 |
| 节奏单调 | 叙事节奏控制 | novel-writing 模块C | 增加快慢交替 |
| 缺乏金句 | 金句重复法 | novel-writing 模块C | 添加标志性台词 |

### 去AI味集成（novel-deslop）

在优化循环中，每次优化后自动运行去AI味：

- Gate A：禁用词替换（查 novel-deslop/references/banned-words.md）
- Gate B：句式去套路（"不是A而是B"、万能状语）
- Gate C：心理描写外化（Show Don't Tell）
- Gate D：节奏打碎（排比、长句）
- Gate E：对话去腔调（角色差异化）
- Gate F：结尾去升华

**执行顺序**：先优化短板 → 再去AI味 → 最后重新评分。详见 `/novel-deslop`。

### 优化流程

```
1. 分析当前正文的短板（得分<7分的维度）
2. 从 novel-writing 中选择对应的优化技法
3. 保留核心情节，针对性增强薄弱环节
4. 输出优化版正文
5. 运行 novel-deslop 去AI味（Gate A-F 扫描 + 三遍法修复）
6. 重新评分，判断是否达标
7. 未达标则继续优化循环
```

# 阶段二：资料支撑

    current_round += 1

    if current_round == 1:
        # 阶段三：初始生成

    else:

    # 记录最优版本

        best_chapter = chapter

    # 判断是否达标
    if current_score >= target_score:
        # 阶段六：深度分析 + 世界观更新
        deep_analysis = analyze_chapter(chapter)
        novel_manage_update(chapter)            # 更新 novel-manage
        output_final_chapter(chapter, score_report, deep_analysis, versions)
        break

    # 输出当轮报告
    output_round_report(chapter, score_report, current_round)

# 达到最大轮数，输出最优版本（标注未达标）
if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_chapter(best_chapter)
    novel_manage_update(best_chapter)
    output_best_chapter(best_chapter, best_score, versions, note="未达标")

## 批量模式完整流程

### 批量编排逻辑

```python
# 伪代码逻辑
batch_size = 10             # 默认每批10章
target_score = 8.5          # 批量模式阈值（低于单章的9.0）
batch_pass_avg = 8.0        # 批次通过的平均分阈值
progress_file = "追踪/进度.md"

# 读取进度文件（断点续写）
if --resume:
    progress = read_progress_file(progress_file)
    start_chapter = progress.last_completed_chapter + 1
else:
    start_chapter = --from or 1

end_chapter = --to or total_chapters
chapters_to_generate = range(start_chapter, end_chapter + 1)

# 按批次分组
batches = split_into_batches(chapters_to_generate, batch_size)

for batch in batches:
    batch_scores = []

    for chapter_num in batch:
        # 读取章节大纲
        outline = read_chapter_outline(chapter_num)

        # === 执行单章6阶段流水线 ===

        # 阶段一：场景构思（自动选择最优方向）
        direction = auto_best_direction(outline)

        # 阶段二：资料支撑
        worldbuilding = novel_manage_read()
        writing_feedback = read_file("追踪/写作反馈.md")  # 读取管理反馈
        research = web_search_if_needed(outline)

        # 阶段三：正文生成
        chapter = novel_writing_generate(outline, direction, research)

        # 阶段四：评分
        score = novel_quality_score(chapter)

        # 阶段五：优化（如果未达标，优化一轮）
        if score < target_score:
            chapter = optimize_chapter(chapter, score.weaknesses)
            chapter = novel_deslop_scan(chapter)
            score = novel_quality_score(chapter)

        # 阶段六：世界观更新
        novel_manage_update(chapter)

        # 保存章节文件
        if score >= target_score:
            save_chapter(chapter, status="完成")
        else:
            save_chapter(chapter, status="需手动优化")

        batch_scores.append(score)

        # 更新进度文件
        update_progress_file(progress_file, chapter_num, score)

    # 批次级一致性检查 + 生成写作反馈
    consistency_report = novel_manage_check(batch)
    # 检查完成后自动更新 追踪/写作反馈.md，供后续章节使用

    # 批次进度报告
    avg_score = average(batch_scores)
    output_batch_report(batch, batch_scores, avg_score)

    # 判断是否继续下一批
    if avg_score < batch_pass_avg:
        warn(f"批次平均分 {avg_score} < {batch_pass_avg}，建议检查后继续")
        # 不自动停止，但发出警告
```

## 创作规则

### 1. 连贯性原则
- **风格连贯**：与前文保持一致的叙事风格
- **情节连贯**：接续前文的情节，不能有断裂
- **角色连贯**：角色核心个性一致，但情绪/行为/语言可随场景变化
- **情感连贯**：关系进展符合前文铺垫

### 1.2 去耦合化组合原则
- **角色表现 = 个性类型×情绪状态×行为动作×语言风格**：同一角色在不同场景可以有不同表现
- **章节内容 = 事件类型×事件风格×场景氛围×角色特点**：通过组合创造惊喜
- **避免单一模式**：不要让"高冷型"角色永远高冷，让她在害羞时咬唇、在愤怒时毒舌、在伤心时沉默
- **核心体验极端化**：本章的核心体验（暧昧/悲伤/搞笑/...）要做到极致

### 1.1 反馈约束原则
- **必须读取反馈**：写作前必须读取 `追踪/写作反馈.md`，将管理检查的反馈纳入写作
- **避免重复犯错**：反馈中的"高频问题"是已知的写作陷阱，必须主动规避
- **遵循角色建议**：反馈中的"角色注意事项"是已发现的角色问题，写作时按建议执行
- **处理伏笔预警**：反馈中的"伏笔预警"提示即将超期的伏笔，优先在后续章节中回收
- **衔接要点**：反馈中的"上一章衔接要点"确保本章与前文无缝衔接

### 2. 字数控制
- 目标字数：4000字
- 允许浮动：+-500字（3500-4500字）
- 过长处理：拆分场景，保留核心内容
- 过短处理：增加细节描写、内心独白

### 3. 吸引力元素植入
- **亲密场景**：根据章节大纲设计，遵循渐进式
- **幽默元素**：每章至少2-3个笑点，自然融入
- **日常细节**：穿插在对话和动作中，不刻意
- **悬念钩子**：每章结尾必须有

### 4. 描写细腻度
- **对话**：有潜台词，有动作穿插，有节奏变化
- **动作**：具体到手指、眼神、微表情
- **环境**：感官叠加，有氛围感
- **内心**：有独白，有情感流动
