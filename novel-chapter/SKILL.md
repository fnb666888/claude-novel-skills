# 小说正文迭代生成系统

**核心理念**：正文不是一次生成的，而是通过"生成→评分→优化"的迭代循环打磨出来的。

**本skill整合全部创作工具**，自动完成全流程。

---

## 整合工具清单

| 工具 | 整合阶段 | 作用 |
|:---|:---|:---|
| **内部头脑风暴** | 阶段一：场景构思 | 探索场景表现方式 |
| **WebSearch** | 阶段二：资料支撑 | 搜索专业细节、背景资料 |
| **novel-worldbuilding** | 阶段二+六 | 读取/更新世界观设定 |
| **novel-writing** | 阶段三：正文生成 | 提供写作技法（亲密/幽默/日常/对话） |
| **novel-evaluation** | 阶段四：体验评分 | 三层层级评价体系（量表C） |
| **novel-optimization** | 阶段五：吸引力优化 | 针对短板的优化技法 |
| **novel-deslop** | 阶段五：去AI味 | 6 Gate扫描 + 三遍法修复 |

> **说明**：本工具已移除伪整合，改为内部实现或真实工具调用。

## 使用方式

### 基础用法
- `/novel-chapter [章节大纲文件路径]` — 根据章节大纲生成正文
- `/novel-chapter [章节大纲文件路径] --previous [上一章文件路径]` — 指定上一章内容

### 进阶用法
- `/novel-chapter [章节大纲文件路径] --focus 亲密` — 本章重点强化亲密描写
- `/novel-chapter [章节大纲文件路径] --focus 幽默` — 本章重点强化幽默元素
- `/novel-chapter [章节大纲文件路径] --focus 日常` — 本章重点强化日常细节
- `/novel-chapter [章节大纲文件路径] --focus 惊喜` — 本章重点强化意外反转
- `/novel-chapter [章节大纲文件路径] --words 5000` — 自定义字数（默认4000）

### 流程控制
- `/novel-chapter --max 3` — 最多迭代3轮（默认5轮）
- `/novel-chapter --target 8.5` — 目标分数8.5（默认9.0）

### 达标标准
- **默认目标分数：9.0**（极品正文）
- **达标判定：总分 ≥ 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

## 阶段一：场景构思（brainstorming）

**工具**：brainstorming（头脑风暴）

**目标**：在正式生成正文前，先进行头脑风暴，探索场景的最佳表现方式。

### 执行流程

```
1. 读取章节大纲，理解本章的核心任务
2. 根据章节类型（开篇/亲密/日常/高潮/转折），进行发散思考
3. 生成3-5个不同的场景表现方向，每个方向包含：
   - 场景切入点（从哪里开始写）
   - 情绪基调（本章的主要情绪）
   - 吸引力元素植入点（亲密/幽默/悬念放在哪里）
   - 潜在风险（可能的问题）
4. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 场景构思头脑风暴

## 章节：第XX章 [章节标题]

## 章节类型：[开篇章/亲密章/日常章/高潮章/转折章]

## 核心任务：[本章要完成什么]

## 场景表现方向

### 方向A：[名称]
- 场景切入点：[从哪里开始]
- 情绪基调：[主要情绪]
- 吸引力植入：[亲密/幽默/悬念位置]
- 潜在风险：[可能的问题]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```

## 阶段三：正文生成（novel-writing）

**工具**：novel-writing（创作技法工具箱 - 模块C）

**目标**：基于场景构思和资料支撑，生成约4000字的高质量正文。

### 4000字结构

| 部分 | 字数 | 功能 | 说明 |
|------|------|------|------|
| 开篇承接 | 300-500字 | 接续前文 | 时间/地点/状态过渡 |
| 场景1 | 800-1200字 | 主要内容 | 剧情推进/角色互动 |
| 场景2 | 800-1200字 | 主要内容 | 亲密/幽默/冲突 |
| 场景3 | 800-1200字 | 主要内容 | 高潮/转折/日常 |
| 收尾钩子 | 300-500字 | 章末悬念 | 驱动追更 |

### 写作技法（来自novel-writing模块C）

#### 开篇承接技法
- 时间过渡法：[时间标志词]，[地点/状态变化]
- 场景延续法：[延续上一章场景]，[自然过渡]
- 内心独白承接法：[角色内心想法]，[引出新场景]

#### 亲密描写技法
- 感官分层法：触觉→温度→深化→反馈→力度
- 微反应刻画法：用身体微反应展现情感
- 渐进式隐喻法：小雨→中雨→大雨→雷鸣
- 克制留白法：在最高潮时突然收住
- 环境烘托法：用场景氛围配合亲密情绪
- 感官差异化法：不同角色不同"味道"

#### 幽默描写技法
- 内心独白式幽默：严肃场景+内心吐槽
- 对比式幽默：两个极端的并置
- 破折号旁白法：用破折号引出旁白
- 逻辑包装法：用歪理为亲密行为找借口
- 误会式幽默：信息错位制造喜剧
- 旁观者呆滞法：局外人的困惑反应

#### 日常细节技法
- 感官叠加法：视觉+听觉+触觉+嗅觉+味觉
- 生活细节法：饮食/穿着/居住/习惯
- 习惯标签法：给角色设计标志性习惯

#### 悬念钩子技法
- 悬念型："这时，XX发生了..."
- 转折型："就在一切顺利的时候，意外发生了。"
- 信息揭露型："他终于知道了那个秘密。"
- 对话截断型："她只说了一个字：'好。'"
- 悬念提问型："如果她知道了一切，会怎样？"

## 阶段四：体验评分（novel-evaluation）

**工具**：novel-evaluation（评价体系 - 量表C）

**目标**：量化评估正文的读者体验，识别短板。

### 评分框架（三层层级）

#### 第一层：章节目标达成度（40%）

根据章节类型，评价该章节是否完成了它的核心任务。

| 章节类型 | 核心任务 | 评价维度 |
|----------|----------|----------|
| 开篇章 | 让读者想看第二章 | 世界观清晰度、角色引入效果、悬念设置强度 |
| 亲密章 | 让读者心动 | 亲密描写质量、情感真实度、余韵处理 |
| 日常章 | 让读者有代入感 | 生活细节真实度、角色互动自然度、感情线推进 |
| 高潮章 | 让读者"卧槽" | 冲突张力、情感爆发力、反转惊喜度 |
| 转折章 | 让读者改变预期 | 信息揭露效果、剧情转向力度、悬念延续性 |

#### 第二层：读者体验维度（40%）

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 沉浸感 | 37.5% | 读者忘记时间，完全沉浸在故事中？ |
| 情绪波动 | 37.5% | 读者经历3种以上情绪变化？ |
| 翻页欲 | 25% | 读者迫不及待想看下一章？ |

#### 第三层：技巧执行度（20%）

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 描写质量 | 30% | 感官描写细腻，有多感官叠加？ |
| 对话质量 | 30% | 对话有潜台词，有动作穿插？ |
| 节奏控制 | 25% | 快慢交替自然，有"减压阀"？ |
| 金句记忆 | 15% | 有让人想截图/引用的句子？ |

### 评分计算

```
总分 = 第一层×40% + 第二层×40% + 第三层×20%

第一层得分 = Σ (维度得分 × 维度权重)
第二层得分 = 沉浸感×37.5% + 情绪波动×37.5% + 翻页欲×25%
第三层得分 = 描写质量×30% + 对话质量×30% + 节奏控制×25% + 金句记忆×15%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **✓ 采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的正文才会被采用并保存为最终文件。**

### craft钩子补充

**工具**：craft（钩子发现）

在优化过程中，使用craft发现新的钩子和吸引力元素：
- 发现新的悬念钩子
- 发现新的亲密描写角度
- 发现新的幽默触发点
- 发现新的金句素材

### 优化流程

```
1. 分析当前正文的短板（得分<7分的维度）
2. 从novel-writing中选择对应的优化技法
3. 使用craft发现新的钩子元素
4. 保留核心情节，针对性增强薄弱环节
5. 输出优化版正文
6. 运行 novel-deslop 去AI味（Gate A-F 扫描 + 三遍法修复）
7. 重新评分，判断是否达标
8. 未达标则继续优化循环
```

### 去AI味集成

**工具**：novel-deslop（去AI味系统）

在优化循环中，每次优化后自动运行去AI味：
- Gate A：禁用词替换（查 novel-deslop/references/banned-words.md）
- Gate B：句式去套路（"不是A而是B"、万能状语）
- Gate C：心理描写外化（Show Don't Tell）
- Gate D：节奏打碎（排比、长句）
- Gate E：对话去腔调（角色差异化）
- Gate F：结尾去升华

去AI味在优化之后、再评分之前执行。详见 `/novel-deslop`。

## 迭代控制逻辑

### 迭代判定规则

```python
# 伪代码逻辑
target_score = 9.0  # 默认目标分数（必须达到9分才能采用）
max_rounds = 5      # 默认最大轮数（正文迭代轮数较少）
current_round = 0
best_score = 0
best_chapter = None

# 读取章节大纲、世界观设定、上一章内容
chapter_outline = read_chapter_outline(章节大纲文件路径)
worldbuilding = read_worldbuilding()
previous_chapter = read_previous_chapter() if --previous else None

# 阶段一：场景构思（内部发散思考）
selected_direction = user_select_or_auto_best(场景表现方向列表)

# 阶段二：资料支撑
research_data = web_search(需要查询的资料)

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        # 阶段三：初始生成
        chapter = novel_writing_generate(chapter_outline, selected_direction, research_data)
    else:
        # 阶段五：定向优化
        chapter = optimize_chapter(chapter, weaknesses)

    # 阶段四：评分
    score_report = novel_evaluation_score(chapter)
    current_score = score_report.total_score

    # 记录最优版本
    if current_score > best_score:
        best_score = current_score
        best_chapter = chapter

    # 判断是否达标
    if current_score >= target_score:
        # 阶段六：深度分析 + 世界观更新
        deep_analysis = analyze_chapter(chapter)
        update_worldbuilding_after_chapter(chapter)
        output_final_chapter(chapter, score_report, deep_analysis, versions)
        break

    # 输出当轮报告
    output_round_report(chapter, score_report, current_round)

# 达到最大轮数，输出最优版本（标注未达标）
if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_chapter(best_chapter)
    update_worldbuilding_after_chapter(best_chapter)
    output_best_chapter(best_chapter, best_score, versions, note="未达标")
```

## 输出文档规则

### 重要：只保留最终达标文件

**本skill只保存最终达标的正文文件（9.0+分），中间迭代过程不保存为文件。**

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：`chapter_[章节号]_[时间戳].md`

### 保存条件
- **只有总分 ≥ 9.0 的正文才会保存为文件**
- 未达标的迭代过程仅在对话中显示，不保存文件
- 达到最大轮数仍未达标时，保存当前最优版本（需标注"未达标"）

## 创作规则

### 1. 连贯性原则
- **风格连贯**：与前文保持一致的叙事风格
- **情节连贯**：接续前文的情节，不能有断裂
- **角色连贯**：角色性格、语言风格保持一致
- **情感连贯**：关系进展符合前文铺垫

### 2. 字数控制
- 目标字数：4000字
- 允许浮动：±500字（3500-4500字）
- 过长处理：拆分场景，保留核心内容
- 过短处理：增加细节描写、内心独白

### 3. 吸引力元素植入
- **亲密场景**：根据章节大纲设计，遵循渐进式
- **幽默元素**：每章至少2-3个笑点，自然融入
- **日常细节**：穿插在对话和动作中，不刻意
- **悬念钩子**：每章结尾必须有

### 4. 描写细腻度
- **对话**：有潜台词，有动作穿插，有节奏变化
- **动作**：具体到手指、眼神、微表情
- **环境**：感官叠加，有氛围感
- **内心**：有独白，有情感流动
