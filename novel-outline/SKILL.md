# 小说大纲双阶段迭代生成系统

**核心理念**：大纲不是一次生成的，而是通过"生成→评分→优化"的迭代循环打磨出来的。全书大纲和章节大纲分两阶段执行，层层递进。

**本skill整合全部创作工具**，自动完成全流程。

---

## 使用方式

### Phase 1：全书大纲

- `/novel-outline` 或 `/novel-outline 全书` — 完整流程（构思→生成→迭代优化）
- `/novel-outline 全书 [类型]` — 指定类型生成（都市重生、玄幻修仙等）
- `/novel-outline 全书 --inspiration [灵感文件路径]` — 基于已有灵感扩展
- `/novel-outline 全书 --words 150万` — 指定目标字数（默认150万）
- `/novel-outline 全书 --focus 亲密` — 重点强化亲密关系设计
- `/novel-outline 全书 --focus 幽默` — 重点强化幽默元素设计

### Phase 2：章节大纲

- `/novel-outline 章节` — 基于全书大纲生成下10章详细大纲（Phase 1完成后自动触发）
- `/novel-outline 章节 --from 第XX章` — 从指定章节开始生成
- `/novel-outline 章节 --current [已写章节文件路径]` — 基于已写内容生成后续10章
- `/novel-outline 章节 --focus 亲密` — 这10章重点强化亲密描写
- `/novel-outline 章节 --focus 幽默` — 这10章重点强化幽默元素
- `/novel-outline 章节 --focus 惊喜` — 这10章重点强化意外反转

### 流程控制（两个阶段通用）

- `--max N` — 最多迭代N轮（Phase 1默认10轮，Phase 2默认5轮）
- `--target N` — 目标分数（默认9.0）

### 达标标准

- **默认目标分数：9.0**（极品大纲）
- **达标判定：总分 ≥ 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

## Phase 1 阶段一：结构构思（brainstorming）

**目标**：在正式生成大纲前，先进行头脑风暴，探索剧情架构方向。

### 执行流程

```
1. 根据用户指定的类型（如有）或灵感文件，进行发散思考
2. 生成3-5个不同的剧情架构方向，每个方向包含：
   - 故事骨架（主线冲突+终极目标）
   - 卷数规划（4-6卷）
   - 核心卖点（为什么读者会追更）
   - 潜在风险（节奏可能的问题）
3. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 剧情架构头脑风暴

## 类型：[用户指定类型 或 基于灵感]

## 架构方向

### 方向A：[名称]
- 故事骨架：[主线冲突+终极目标]
- 卷数规划：[X卷]
- 核心卖点：[最吸引人的点]
- 潜在风险：[节奏问题]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```

## Phase 1 阶段三：大纲生成（novel-writing 去耦合化组合）

**目标**：基于头脑风暴和资料支撑，通过去耦合化组合生成充满惊喜的150万字容量大纲。

### 大纲框架（八大部分）

#### 第一部分：核心设定
- 一句话定义、类型标签、目标读者、核心爽点
- **核心体验类型声明**（暧昧型/搞笑型/悬疑型/热血型/甜蜜型/...）
- 世界观设定、金手指、势力分布、等级体系

#### 第二部分：角色设计（去耦合化组合）
- 主角设计：从 character-traits.md 选取 **个性类型×情绪状态×行为动作×语言风格**
- 女性角色矩阵4-6位：每位用不同组合，确保差异化和惊喜感
  - **不是**：选一个"高冷型"按套路写
  - **而是**：高冷型×害羞情绪×咬唇行为×惜字如金语言 → 组合本身就是惊喜
- 配角设计5-8位：同样用去耦合化组合
- 结合 extensions/ 中的扩展角色特点（如有）

#### 第三部分：主线剧情（去耦合化组合）
- 主线概述（核心冲突、终极目标）
- 剧情弧设计5-7个：每个弧的核心情节用 **事件类型×事件风格×场景氛围** 组合
  - **不是**：选一个"误会"按模板写
  - **而是**：误会冲突×心动甜蜜风格×暧昧私密氛围 → 误会也能甜到牙疼
- 结合 extensions/ 中的扩展情节元素（如有）

#### 第四部分：感情线设计
- 感情线总览（4-6条）
- 每条感情线详细设计（四阶段、亲密场景、感官差异化）
- 感情线交织设计（修罗场、冲突、和解）
- **每条感情线的核心体验不同**（如：第一条甜蜜型、第二条暧昧型、第三条虐心型）

#### 第五部分：吸引力元素
- 亲密关系设计（场景数量、类型分布、禁忌感、克制突破）
- 日常细节设计（场景数量、烟火气细节、感官细节）
- 幽默元素设计（场景数量、类型分布、金句、破折号旁白）
- 惊喜反转设计（反转数量、类型、信息差、表里双面）
- 悬念钩子设计（钩子类型、核心悬念线、阶段性悬念）
- **核心体验极端化设计**：围绕核心体验类型，设计将其推到极致的元素

#### 第六部分：节奏控制
- 整体节奏设计（高压/低压比例、多线交织比例）
- 吸引力节奏图（亲密/幽默/悬念/情感峰值分布）

#### 第七部分：支线剧情
- 支线列表（3-5条）
- 支线与主线的交织

#### 第八部分：字数分配总览
- 按内容类型分配
- 按剧情弧分配
- 按吸引力元素分配

## Phase 1 阶段四：节奏评分（novel-quality 量表B 极端化评估）

**目标**：量化评估大纲的节奏设计，识别核心体验是否做到极致。

### 评分流程

```
1. 声明核心体验类型（如"甜宠型大纲"→ 亲密节奏为核心维度）
2. 评估7个维度的得分
3. 检查核心体验维度是否 ≥ 9.5
4. 应用极端化修正（核心维度≥9.5则总分上浮）
5. 判断是否达标
```

### 评分维度（量表B：全书大纲评价）

| 维度 | 权重 | 核心问题 | 评分标准 |
|------|------|----------|----------|
| **亲密节奏** | 30% | 亲密场景分布是否合理？有渐进式升级？ | 9-10:渐进升级 7-8:有阶段 5-6:一般 3-4:生硬 1-2:几乎没有 |
| **日常密度** | 15% | 有多少让人会心一笑的日常场景？ | 9-10:每3-5章 7-8:合理 5-6:偏少 3-4:几乎没有 1-2:无 |
| **幽默分布** | 15% | 笑点是否均匀分布？有"减压阀"？ | 9-10:每2-3章 7-8:合理 5-6:偏少 3-4:生硬 1-2:无 |
| **惊喜节点** | 15% | 在哪些位置设置了出人意料的转折？ | 9-10:20%/50%/80% 7-8:2-3个 5-6:少 3-4:缺乏 1-2:无 |
| **节奏控制** | 10% | 快慢交替是否合理？有张弛？ | 9-10:自然交替 7-8:基本合理 5-6:一般 3-4:单调 1-2:无 |
| **悬念密度** | 10% | 有多少"想知道后续"的钩子？ | 9-10:每章都有 7-8:大部分有 5-6:偏少 3-4:稀疏 1-2:无 |
| **情感峰值** | 5% | 是否有让人泪目/心动的高潮点？ | 9-10:2-3个分布合理 7-8:有 5-6:弱 3-4:缺乏 1-2:无 |

### 评分计算

```
总分 = 亲密节奏×30% + 日常密度×15% + 幽默分布×15% + 惊喜节点×15% + 节奏控制×10% + 悬念密度×10% + 情感峰值×5%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的大纲才会被采用并保存为最终文件。**

## Phase 1 阶段五：节奏优化（novel-quality 极端化优先）

**目标**：先将核心体验推到极致，再修补明显短板。

### 优化策略（极端化优先）

**第一步：核心体验极端化**
- 确定大纲的核心体验类型（甜宠/搞笑/悬疑/热血/治愈/...）
- 集中火力将核心体验维度推到 9.5+
- 使用去耦合化组合增强核心体验的独特性

**第二步：短板修补（仅修补 < 5 分的维度）**
- 只修补会严重影响读者体验的明显短板
- 不追求所有维度都高分

#### 亲密节奏极端化（甜宠型大纲）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 分布不均 | 渐进式隐喻系统 | 模块C | 设计小雨→中雨→大雨→雷鸣的递进 |
| 缺乏升级感 | 渐进式推进法 | 模块B | 设计四阶段亲密递进 |
| 缺乏禁忌感 | 身份禁忌极致化 | 模块C | 设计"绝对不能碰"的禁忌线 |
| 缺乏克制 | 点到为止的克制 | 模块C | 设计"差一点"的克制场景 |

#### 日常密度优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 日常场景少 | 场景构建 | 模块C | 增加居家/外出/节日场景 |
| 缺乏烟火气 | 感官叠加法 | 模块C | 添加视觉/听觉/触觉细节 |
| 缺乏生活感 | 生活细节法 | 模块C | 设计吃饭/逛街/居家场景 |

#### 幽默分布优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 笑点少 | 幽默技巧 | 模块C | 添加内心独白式幽默 |
| 分布不均 | 破折号旁白法 | 模块C | 在高压段后插入减压阀 |
| 类型单一 | 对比式幽默 | 模块C | 设计多种幽默类型交替 |

#### 惊喜节点优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 反转少 | 信息差驱动法 | 模块B | 设计读者vs角色的信息差 |
| 位置不当 | 钩子设置技巧 | 模块B | 在20%/50%/80%位置设置反转 |
| 缺乏深度 | 表里双层叙事法 | 模块A | 设计表里不一的角色揭露 |

#### 节奏控制优化（权重10%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 节奏单调 | 叙事节奏控制 | 模块C | 设计快慢交替的节奏 |
| 缺乏减压阀 | 幽默技巧 | 模块C | 在紧张后插入轻松场景 |
| 缺乏张弛 | 高潮场景设计 | 模块B | 设计高压段和低压段交替 |

#### 悬念密度优化（权重10%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 钩子少 | 钩子设置技巧 | 模块B | 每章结尾设计钩子 |
| 类型单一 | 信息差驱动法 | 模块B | 设计多种钩子类型交替 |
| 缺乏驱动力 | 嵌套式叙事框架 | 模块B | 设计贯穿全文的悬念线 |

#### 情感峰值优化（权重5%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 高潮少 | 高潮场景设计 | 模块B | 设计2-3个情感高潮点 |
| 位置不当 | 以死换情法 | 模块B | 在30%/60%/90%位置设置高潮 |
| 冲击力弱 | 重复递进情绪法 | 模块C | 用重复句式递进情绪强度 |

### 钩子补充

在优化过程中，主动发现新的钩子和吸引力元素：
- 发现新的悬念钩子
- 发现新的亲密节奏设计
- 发现新的幽默触发点
- 发现新的反转设置方式

### 优化流程

```
1. 分析当前大纲的短板（得分<7分的维度）
2. 从novel-writing中选择对应的优化技法
3. 发现新的钩子元素
4. 保留核心架构，针对性增强薄弱环节
5. 输出优化版大纲
6. 重新评分，判断是否达标
7. 未达标则继续优化循环
```

## Phase 1 迭代控制逻辑

```python
# 伪代码逻辑（实际执行时由Claude内部实现）
target_score = 9.0
max_rounds = 10
current_round = 0
best_score = 0
best_outline = None
versions = []

# 读取灵感文件（如有）
inspiration = read_inspiration_file() if --inspiration else None
# 读取世界观设定

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        outline = generate_outline(selected_direction, research_data, inspiration, core_experience_type)
    else:
        outline = optimize_outline(outline, weaknesses, core_experience_type)

    score_report = evaluate_outline(outline, core_experience_type)  # novel-quality 量表B + 极端化评估
    current_score = score_report.total_score  # 含极端化修正

    versions.append({round: current_round, score: current_score, content: outline, core_score: score_report.core_dimension_score})

    if current_score > best_score:
        best_score = current_score
        best_outline = outline

    # 判断是否达标（总分≥9.0 或 核心体验≥9.5且其他≥5）
    if current_score >= target_score or score_report.extreme_pass:
        deep_analysis = analyze_outline(outline)
        expand_worldbuilding(outline)  # novel-manage 扩展世界观
        output_final_outline(outline, score_report, deep_analysis, versions)
        break

    output_round_report(outline, score_report, current_round)

if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_outline(best_outline)
    expand_worldbuilding(best_outline)
    output_best_outline(best_outline, best_score, versions, note="未达标")
```

## Phase 2 阶段三：大纲生成（novel-writing 去耦合化组合）

**目标**：基于节奏构思和资料支撑，通过去耦合化组合生成充满惊喜的10章详细大纲。

### 去耦合化组合原则

**每章的情节设计都使用去耦合化组合**：
- 从 character-traits.md 选取角色在该场景的表现组合
- 从 plot-elements.md 选取事件类型×事件风格×场景氛围
- 确保10章内的组合不重复，每章都有独特惊喜

### 10章定位

10章 ≈ 2-3万字，是一个**中型叙事单元**：
- 不需要完成一个完整故事弧
- 但需要有**起承转合**的节奏
- 要有**承上启下**的功能
- 要有**至少1-2个高潮点**

| 第8章 | 高潮 | 亲密/情感高潮 | 大雨级别/情感爆发 |
| 第9章 | 转折 | 惊喜反转 | 出人意料的发展 |
| 第10章 | 启下 | 悬念钩子 | 章末钩子，驱动追更 |

| 元素 | 最低数量 | 目标数量 | 说明 |
|------|----------|----------|------|

| 悬念钩子 | 5个 | 10个 | 每章结尾都要有 |

## Phase 2 阶段四：节奏评分（novel-quality 章节大纲评分 极端化评估）

**目标**：量化评估章节大纲的节奏设计，识别核心体验是否做到极致。

### 评分流程

```
1. 声明这10章的核心体验类型
2. 评估6个维度的得分
3. 检查核心体验维度是否 ≥ 9.5
4. 应用极端化修正
5. 判断是否达标
```

### 评分维度（章节大纲评价）

| 维度 | 权重 | 核心问题 | 评分标准 |
|------|------|----------|----------|
| **节奏完整性** | 25% | 10章是否有完整的起承转合？ | 9-10:完美起承转合 7-8:基本完整 5-6:一般 3-4:不完整 1-2:无节奏 |
| **亲密节奏** | 25% | 亲密场景分布是否合理？有渐进升级？ | 9-10:渐进升级 7-8:有阶段 5-6:一般 3-4:生硬 1-2:无 |
| **悬念密度** | 20% | 每章是否有钩子？能否驱动追更？ | 9-10:每章都有 7-8:大部分有 5-6:偏少 3-4:稀疏 1-2:无 |
| **幽默分布** | 15% | 笑点是否均匀分布？有"减压阀"？ | 9-10:均匀分布 7-8:合理 5-6:偏少 3-4:生硬 1-2:无 |
| **承上启下** | 10% | 是否能接续前文并为后续铺垫？ | 9-10:完美衔接 7-8:基本连贯 5-6:一般 3-4:有断裂 1-2:脱节 |
| **场景详细度** | 5% | 重点场景是否有足够细节？ | 9-10:非常详细 7-8:详细 5-6:一般 3-4:粗略 1-2:无细节 |

### 评分计算

```
总分 = 节奏完整性×25% + 亲密节奏×25% + 悬念密度×20% + 幽默分布×15% + 承上启下×10% + 场景详细度×5%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的章节大纲才会被采用并保存为最终文件。**

## Phase 2 迭代控制逻辑

```python

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        # 阶段三：初始生成
        chapter_outline = generate_chapter_outline(
            overall_outline, selected_direction, research_data, previous_batch
        )
    else:
        # 阶段五：定向优化

    score_report = evaluate_chapter_outline(chapter_outline)
    current_score = score_report.total_score

    # 记录最优版本
    if current_score > best_score:
        best_score = current_score
        best_outline = chapter_outline

    # 判断是否达标
    if current_score >= target_score:
        # 阶段六：深度分析 + 世界观更新
        deep_analysis = analyze_chapter_outline(chapter_outline)
        update_worldbuilding(chapter_outline)  # novel-manage 更新伏笔追踪
        output_final_chapter_outline(chapter_outline, score_report, deep_analysis)
        break

    # 输出当轮报告
    output_round_report(chapter_outline, score_report, current_round)

# 达到最大轮数，输出最优版本（标注未达标）
if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_chapter_outline(best_outline)
    update_worldbuilding(best_outline)
    output_best_chapter_outline(best_outline, best_score, note="未达标")
```

## Phase 2 大纲生成规则

### 1. 承上启下原则

- **承上**：第1章必须接续前文，不能有断裂感
- **启下**：第10章必须为后续铺垫，不能有完结感
- **连续性**：角色状态、情感进展、剧情线索必须连贯
- **首批次特殊处理**：第1批（第1-10章）无需承上，但必须启下

### 2. 吸引力元素密度

- **亲密场景**：至少2个，目标3-4个，必须有渐进升级
- **笑点**：至少3个，目标5-8个，多种类型交替
- **日常场景**：至少2个，目标3-4个，有烟火气
- **悬念钩子**：每章结尾都要有
- **惊喜反转**：可选，1-2个

### 3. 节奏控制

- **张弛有度**：高压段后必有低压段
- **减压阀**：紧张/冲突后必须有轻松/幽默
- **渐进升级**：亲密/冲突都要逐步升级

### 4. 技法运用

- 亲密场景：参照novel-writing模块C的亲密场景写作技法
- 幽默场景：参照novel-writing模块C的幽默技巧
- 日常场景：参照novel-writing模块C的场景构建
- 悬念钩子：参照novel-writing模块B的钩子设置技巧
