# 小说大纲迭代生成系统

**核心理念**：大纲不是一次生成的，而是通过"生成→评分→优化"的迭代循环打磨出来的。

**本skill整合全部创作工具**，自动完成全流程。

---

## 整合工具清单

| 工具 | 整合阶段 | 作用 |
|:---|:---|:---|
| **内部头脑风暴** | 阶段一：结构构思 | 探索剧情架构方向 |
| **WebSearch** | 阶段二：资料支撑 | 搜索背景资料、职业设定、历史事件 |
| **novel-worldbuilding** | 阶段二+六 | 读取/扩展世界观设定 |
| **novel-writing** | 阶段三：大纲生成 | 提供角色类型、情节模板、感情线设计 |
| **novel-evaluation** | 阶段四：节奏评分 | 7维度评分体系（量表B） |
| **novel-optimization** | 阶段五：节奏优化 | 针对短板的优化技法 |

> **说明**：本工具已移除伪整合（brainstorming/context7/notepad/craft/brain-methodology），改为内部实现或真实工具调用。

## 使用方式

### 基础用法
- `/novel-outline` — 完整流程（构思→生成→迭代优化）
- `/novel-outline [类型]` — 指定类型生成（都市重生、玄幻修仙等）
- `/novel-outline --inspiration [灵感文件路径]` — 基于已有灵感扩展

### 进阶用法
- `/novel-outline --words 150万` — 指定目标字数（默认150万）
- `/novel-outline --focus 亲密` — 重点强化亲密关系设计
- `/novel-outline --focus 幽默` — 重点强化幽默元素设计

### 流程控制
- `/novel-outline --max 5` — 最多迭代5轮（默认10轮）
- `/novel-outline --target 8.5` — 目标分数8.5（默认9.0）

### 达标标准
- **默认目标分数：9.0**（极品大纲）
- **达标判定：总分 ≥ 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

## 阶段一：结构构思（brainstorming）

**工具**：brainstorming（头脑风暴）

**目标**：在正式生成大纲前，先进行头脑风暴，探索剧情架构方向。

### 执行流程

```
1. 根据用户指定的类型（如有）或灵感文件，进行发散思考
2. 生成3-5个不同的剧情架构方向，每个方向包含：
   - 故事骨架（主线冲突+终极目标）
   - 卷数规划（4-6卷）
   - 核心卖点（为什么读者会追更）
   - 潜在风险（节奏可能的问题）
3. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 剧情架构头脑风暴

## 类型：[用户指定类型 或 基于灵感]

## 架构方向

### 方向A：[名称]
- 故事骨架：[主线冲突+终极目标]
- 卷数规划：[X卷]
- 核心卖点：[最吸引人的点]
- 潜在风险：[节奏问题]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```

## 阶段二：资料支撑（WebSearch + novel-worldbuilding）

**工具**：WebSearch（网络搜索）+ novel-worldbuilding（世界观管理）

**目标**：查询相关背景资料，为大纲提供真实感和深度。

### 执行流程

```
1. 读取novel-worldbuilding中的世界观设定（如有）
2. 根据选定的架构方向，确定需要查询的资料类型
3. 使用WebSearch搜索需要补充的背景资料
4. 将关键资料点记录到世界观文档中
```

## 阶段三：大纲生成（novel-writing）

**工具**：novel-writing（创作技法工具箱）

**目标**：基于头脑风暴和资料支撑，生成完整的150万字容量大纲。

### 大纲框架

#### 第一部分：核心设定
- 一句话定义、类型标签、目标读者、核心爽点
- 世界观设定、金手指、势力分布、等级体系

#### 第二部分：角色设计
- 主角设计（参照novel-writing模块A：核心六要素）
- 女性角色矩阵4-6位（参照模块A：角色类型速查表）
- 配角设计5-8位（参照模块C：七种配角类型）

#### 第三部分：主线剧情
- 主线概述（核心冲突、终极目标）
- 剧情弧设计5-7个（每个弧：字数、冲突、高潮、钩子）

#### 第四部分：感情线设计
- 感情线总览（4-6条）
- 每条感情线详细设计（四阶段、亲密场景、感官差异化）
- 感情线交织设计（修罗场、冲突、和解）

#### 第五部分：吸引力元素
- 亲密关系设计（场景数量、类型分布、禁忌感、克制突破）
- 日常细节设计（场景数量、烟火气细节、感官细节）
- 幽默元素设计（场景数量、类型分布、金句、破折号旁白）
- 惊喜反转设计（反转数量、类型、信息差、表里双面）
- 悬念钩子设计（钩子类型、核心悬念线、阶段性悬念）

#### 第六部分：节奏控制
- 整体节奏设计（高压/低压比例、多线交织比例）
- 吸引力节奏图（亲密/幽默/悬念/情感峰值分布）

#### 第七部分：支线剧情
- 支线列表（3-5条）
- 支线与主线的交织

#### 第八部分：字数分配总览
- 按内容类型分配
- 按剧情弧分配
- 按吸引力元素分配

## 阶段四：节奏评分（novel-evaluation）

**工具**：novel-evaluation（评价体系 - 量表B）

**目标**：量化评估大纲的节奏设计，识别短板。

### 评分维度（量表B：大纲评价）

| 维度 | 权重 | 核心问题 | 评分标准 |
|------|------|----------|----------|
| **亲密节奏** | 30% | 亲密场景分布是否合理？有渐进式升级？ | 9-10:渐进升级 7-8:有阶段 5-6:一般 3-4:生硬 1-2:几乎没有 |
| **日常密度** | 15% | 有多少让人会心一笑的日常场景？ | 9-10:每3-5章 7-8:合理 5-6:偏少 3-4:几乎没有 1-2:无 |
| **幽默分布** | 15% | 笑点是否均匀分布？有"减压阀"？ | 9-10:每2-3章 7-8:合理 5-6:偏少 3-4:生硬 1-2:无 |
| **惊喜节点** | 15% | 在哪些位置设置了出人意料的转折？ | 9-10:20%/50%/80% 7-8:2-3个 5-6:少 3-4:缺乏 1-2:无 |
| **节奏控制** | 10% | 快慢交替是否合理？有张弛？ | 9-10:自然交替 7-8:基本合理 5-6:一般 3-4:单调 1-2:无 |
| **悬念密度** | 10% | 有多少"想知道后续"的钩子？ | 9-10:每章都有 7-8:大部分有 5-6:偏少 3-4:稀疏 1-2:无 |
| **情感峰值** | 5% | 是否有让人泪目/心动的高潮点？ | 9-10:2-3个分布合理 7-8:有 5-6:弱 3-4:缺乏 1-2:无 |

### 评分计算

```
总分 = 亲密节奏×30% + 日常密度×15% + 幽默分布×15% + 惊喜节点×15% + 节奏控制×10% + 悬念密度×10% + 情感峰值×5%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **✓ 采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的大纲才会被采用并保存为最终文件。**

## 阶段五：节奏优化（novel-optimization）

**工具**：novel-optimization（优化策略）

**目标**：针对评分短板进行定向优化，补充新的钩子元素。

### 优化策略（根据短板选择）

#### 亲密节奏优化（权重30%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 分布不均 | 渐进式隐喻系统 | 模块C | 设计小雨→中雨→大雨→雷鸣的递进 |
| 缺乏升级感 | 渐进式推进法 | 模块B | 设计四阶段亲密递进 |
| 缺乏禁忌感 | 身份禁忌极致化 | 模块C | 设计"绝对不能碰"的禁忌线 |
| 缺乏克制 | 点到为止的克制 | 模块C | 设计"差一点"的克制场景 |

#### 日常密度优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 日常场景少 | 场景构建 | 模块C | 增加居家/外出/节日场景 |
| 缺乏烟火气 | 感官叠加法 | 模块C | 添加视觉/听觉/触觉细节 |
| 缺乏生活感 | 生活细节法 | 模块C | 设计吃饭/逛街/居家场景 |

#### 幽默分布优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 笑点少 | 幽默技巧 | 模块C | 添加内心独白式幽默 |
| 分布不均 | 破折号旁白法 | 模块C | 在高压段后插入减压阀 |
| 类型单一 | 对比式幽默 | 模块C | 设计多种幽默类型交替 |

#### 惊喜节点优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 反转少 | 信息差驱动法 | 模块B | 设计读者vs角色的信息差 |
| 位置不当 | 钩子设置技巧 | 模块B | 在20%/50%/80%位置设置反转 |
| 缺乏深度 | 表里双层叙事法 | 模块A | 设计表里不一的角色揭露 |

#### 节奏控制优化（权重10%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 节奏单调 | 叙事节奏控制 | 模块C | 设计快慢交替的节奏 |
| 缺乏减压阀 | 幽默技巧 | 模块C | 在紧张后插入轻松场景 |
| 缺乏张弛 | 高潮场景设计 | 模块B | 设计高压段和低压段交替 |

#### 悬念密度优化（权重10%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 钩子少 | 钩子设置技巧 | 模块B | 每章结尾设计钩子 |
| 类型单一 | 信息差驱动法 | 模块B | 设计多种钩子类型交替 |
| 缺乏驱动力 | 嵌套式叙事框架 | 模块B | 设计贯穿全文的悬念线 |

#### 情感峰值优化（权重5%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 高潮少 | 高潮场景设计 | 模块B | 设计2-3个情感高潮点 |
| 位置不当 | 以死换情法 | 模块B | 在30%/60%/90%位置设置高潮 |
| 冲击力弱 | 重复递进情绪法 | 模块C | 用重复句式递进情绪强度 |

### craft钩子补充

**工具**：craft（钩子发现）

在优化过程中，使用craft发现新的钩子和吸引力元素：
- 发现新的悬念钩子
- 发现新的亲密节奏设计
- 发现新的幽默触发点
- 发现新的反转设置方式

### 优化流程

```
1. 分析当前大纲的短板（得分<7分的维度）
2. 从novel-writing中选择对应的优化技法
3. 使用craft发现新的钩子元素
4. 保留核心架构，针对性增强薄弱环节
5. 输出优化版大纲
6. 重新评分，判断是否达标
7. 未达标则继续优化循环
```

## 迭代控制逻辑

### 迭代判定规则

```python
# 伪代码逻辑（实际执行时由Claude内部实现）
target_score = 9.0
max_rounds = 10
current_round = 0
best_score = 0
best_outline = None
versions = []

# 读取灵感文件（如有）
inspiration = read_inspiration_file() if --inspiration else None
# 读取世界观设定
worldbuilding = read_worldbuilding()

# 阶段一：头脑风暴（内部发散思考）
selected_direction = user_select_or_auto_best(架构方向列表)

# 阶段二：资料支撑
research_data = web_search(需要查询的资料)

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        outline = generate_outline(selected_direction, research_data, inspiration)
    else:
        outline = optimize_outline(outline, weaknesses)

    score_report = evaluate_outline(outline)
    current_score = score_report.total_score

    versions.append({round: current_round, score: current_score, content: outline})

    if current_score > best_score:
        best_score = current_score
        best_outline = outline

    if current_score >= target_score:
        deep_analysis = analyze_outline(outline)
        expand_worldbuilding(outline)  # 扩展世界观文档
        output_final_outline(outline, score_report, deep_analysis, versions)
        break

    output_round_report(outline, score_report, current_round)

if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_outline(best_outline)
    expand_worldbuilding(best_outline)
    output_best_outline(best_outline, best_score, versions, note="未达标")
```

## 输出文档规则

### 重要：只保留最终达标文件

**本skill只保存最终达标的大纲文件（9.0+分），中间迭代过程不保存为文件。**

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：`outline_[类型]_[时间戳].md`

### 保存条件
- **只有总分 ≥ 9.0 的大纲才会保存为文件**
- 未达标的迭代过程仅在对话中显示，不保存文件
- 达到最大轮数仍未达标时，保存当前最优版本（需标注"未达标"）

### 输出内容
最终文件包含：
1. 完整大纲（达标版，八大部分）
2. 最终评分报告
3. 深度分析报告
4. 迭代历程总结（每轮分数变化，仅文字记录）

### 输出确认
保存文件后，向用户报告：
- 文件保存路径
- 迭代轮次和最终分数
- 一句话概念
- 主角和主要女性角色列表
- 预计总字数和章节数
- 核心卖点（3个以内）

## 大纲生成规则

### 1. 字数支撑要求
大纲必须设计足够的情节线支撑150万字：
- 主线剧情：至少5-7个大型剧情弧
- 感情线：至少4-6条，每条有完整的发展阶段
- 支线剧情：至少3-5条，与主线有交汇
- 日常生活：贯穿全文，每3-5章一次

### 2. 吸引力元素密度
- 亲密场景：每5-8章一次，有渐进式升级
- 笑点：每2-3章一次，多种类型交替
- 惊喜反转：每20-30章一次，出人意料
- 悬念钩子：每章结尾，驱动追更
- 日常场景：每3-5章一次，有烟火气

### 3. 角色差异化
- 女性角色：4-6个，类型不重复
- 亲密模式：每个角色有独特体验
- 语言风格：不看名字就能区分
- 家庭背景：制造不同相处模式

### 4. 技法引用要求
大纲中使用的技法需注明来源（novel-writing模块A/B/C）。
