# 小说创作项目

## 使用 Skill

本项目使用 `/novel` skill 进行创作。所有写作规则、技法、去AI味规则由 skill 提供。

**触发方式**：`/novel`、`/学习`、`/写作`、`/灵感`、`/大纲`、`/正文`

---

## 会话恢复流程

以下场景需要执行恢复流程：
- 用户说"继续写"
- 开启新会话
- 启动子 Agent 执行创作任务

按以下步骤执行：

### 第一步：理解 Skill

调用 `/novel` 进入创作系统，了解可用的模式和功能。

### 第二步：扫描项目目录

读取项目目录，了解已有内容：

```
项目根目录/
├── 灵感/          → 灵感是否已完成？
├── 设定/          → 世界观是否已建立？
├── 大纲/          → 大纲是否已完成？
│   └── 章节大纲/  → 章节大纲是否已完成？
├── 正文/          → 正文写到第几章？
│   └── [小说名]/  → 各章 txt 文件
└── 追踪/          → 上下文摘要和写作反馈
```

**判断进度的依据**：
- `追踪/上下文摘要.md` → **主要依据**：明确记录当前阶段和进度
- 目录中存在哪些文件 → 辅助验证
- 正文目录中的文件数量 → 辅助验证
- `追踪/写作反馈.md` → 最近的检查反馈

### 第三步：继续创作

根据扫描结果，从断点继续：
- 如果灵感未完成 → 继续灵感阶段
- 如果大纲未完成 → 继续大纲阶段
- 如果正文进行中 → 读取上下文摘要，继续写下一章

## 目录结构说明

| 目录 | 内容 | 格式 |
|------|------|------|
| `灵感/` | 灵感最终版 | md |
| `设定/` | 世界观设定 | md |
| `大纲/` | 全书大纲 + 角色系统 | md |
| `大纲/章节大纲/` | 章节大纲 + 世界线系统 + 角色剧情 | md |
| `正文/[小说名]/` | 各章正文 | txt（纯正文） |
| `追踪/` | 上下文摘要 + 写作反馈 | md |

---

## 关键文件

| 文件 | 用途 |
|------|------|
| `追踪/上下文摘要.md` | 快速恢复上下文（伏笔、关系、当前场景） |
| `追踪/写作反馈.md` | 检查反馈（遗留问题、优化建议） |
| `大纲/角色系统_[小说名].md` | 角色当前状态、关系网络、成长弧线 |
| `大纲/章节大纲/世界线系统_[小说名].md` | 时间线、已发生事件、伏笔状态 |

---

## 工具和规则位置

所有工具和规则由 `/novel` skill 自动加载，无需手动读取：

- **技法库**：`tools/techniques/`（角色塑造、情节构造、正文写作、反面教材）
- **去耦合组件**：`tools/decoupled/`（角色特点分类、情节元素分类）
- **规则集**：`rules/`（输出规则、评分规则、去AI味规则、检查规则）
- **参考资料**：`references/`（优秀小说，用于对比优化）
