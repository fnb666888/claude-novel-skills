# 输出文档规则

**重要：每次运行本skill后，必须将生成的创作内容保存为文档。**

## 文件系统分类

```
项目根目录/
├── CLAUDE.md                          # 静态指引文件（自动加载，教 Claude 使用 skill）
│
├── 灵感/                              # 灵感最终版
│   └── 灵感_[小说名]_最终版.md
│
├── 大纲/                              # 大纲相关
│   ├── 大纲_[小说名]_最终版.md         # 全书大纲最终版
│   ├── 角色系统_[小说名].md            # 角色系统
│   └── 章节大纲/                       # 章节大纲
│       ├── 章节大纲_[小说名]_最终版.md
│       ├── 世界线系统_[小说名].md
│       └── 角色剧情_[小说名].md
│
├── 正文/                              # 正文（纯txt，无评价内容）
│   └── [小说名]/
│       ├── 第001章_[章节名].txt
│       ├── 第002章_[章节名].txt
│       └── ...
│
├── 设定/                              # 世界设定（灵感阶段创建）
│   └── world_[小说名].md
│
└── 追踪/                              # 写作反馈（自动）
    ├── 写作反馈.md                     # 检查反馈（每批更新）
    └── 上下文摘要.md                   # 详细状态（每章/每批更新）
```

## 输出格式规则

## CLAUDE.md 说明

`CLAUDE.md` 是**静态指引文件**，灵感阶段从 skill 目录复制到项目根目录。内容固定，不需要随项目进度更新。

**作用**：教 Claude 如何使用 `/novel` skill 和扫描项目目录判断进度。

**工作原理**：
1. CLAUDE.md 在会话开始时自动加载
2. Claude 按指引调用 `/novel` skill
3. Claude 扫描项目目录（灵感/大纲/正文/追踪）判断当前进度
4. 从断点继续创作

**项目进度**通过目录结构体现，无需在 CLAUDE.md 中记录：
- `灵感/` 有文件 → 灵感已完成
- `正文/` 有 N 个文件 → 正文写到第 N 章
- `追踪/上下文摘要.md` → 最近的上下文信息

| 内容类型 | 格式 | 说明 |
|----------|------|------|
| 创作设定 | `.md` | 创作方向、风格、篇幅等用户选择 |
| 灵感 | `.md` | 只输出最终版 |
| 大纲（全书） | `.md` | 最终版 + 角色系统 |
| 章节大纲 | `.md` | 最终版 + 世界线系统 + 角色剧情 |
| 正文 | `.txt` | 纯正文，不含任何评价、优化、注释内容 |
| 角色系统 | `.md` | 角色档案 |
| 世界线系统 | `.md` | 世界线进程追踪 |
| 设定 | `.md` | 世界观文档 |

## 创作设定模板

首次创作时，将用户选择记录到 `设定/创作设定_[小说名].md`。

```markdown
# 创作设定 — [小说名]

## 基本信息
- 题材类型：[都市言情/古代言情/玄幻修仙/悬疑推理/科幻末世/校园青春/历史架空/其他]
- 创作风格：[轻松幽默/甜宠撒糖/虐心催泪/热血燃向/暗黑悬疑/治愈温馨/其他]
- 计划篇幅：[短篇(1-5万)/中篇(5-20万)/长篇(20-50万)/超长篇(50万+)]
- 每章字数：[2000/3000/4000/5000+]
- 计划章数：[根据篇幅和每章字数计算]

## 核心体验
- 主要体验：[从8种类型中选择]
- 极端化方向：[具体标准]

## 补充说明
[用户其他要求或偏好]
```
