# 输出文档规则

**重要：每次运行本skill后，必须将生成的创作内容保存为文档。**

## 文件系统分类

```
项目根目录/
├── CLAUDE.md                          # 静态指引文件（自动加载，教 Claude 使用 skill）
│
├── 灵感/                              # 灵感最终版
│   └── 灵感_[小说名]_最终版.md
│
├── 大纲/                              # 大纲相关
│   ├── 大纲_[小说名]_最终版.md         # 全书大纲最终版
│   ├── 角色系统_[小说名].md            # 角色系统
│   └── 章节大纲/                       # 章节大纲
│       ├── 章节大纲_[小说名]_最终版.md
│       ├── 世界线系统_[小说名].md
│       └── 角色剧情_[小说名].md
│
├── 正文/                              # 正文（纯txt，无评价内容）
│   └── [小说名]/
│       ├── 第001章_[章节名].txt
│       ├── 第002章_[章节名].txt
│       └── ...
│
├── 设定/                              # 世界设定（灵感阶段创建）
│   └── world_[小说名].md
│
└── 追踪/                              # 写作反馈（自动）
    ├── 写作反馈.md                     # 检查反馈（每批更新）
    └── 上下文摘要.md                   # 详细状态（每章/每批更新）
```

## 输出格式规则

## CLAUDE.md 说明

`CLAUDE.md` 是**静态指引文件**，灵感阶段从 skill 目录复制到项目根目录。内容固定，不需要随项目进度更新。

**作用**：教 Claude 如何使用 `/novel` skill 和扫描项目目录判断进度。

**工作原理**：
1. CLAUDE.md 在会话开始时自动加载
2. Claude 按指引调用 `/novel` skill
3. Claude 扫描项目目录（灵感/大纲/正文/追踪）判断当前进度
4. 从断点继续创作

**项目进度**通过目录结构体现，无需在 CLAUDE.md 中记录：
- `灵感/` 有文件 → 灵感已完成
- `正文/` 有 N 个文件 → 正文写到第 N 章
- `追踪/上下文摘要.md` → 最近的上下文信息

| 内容类型 | 格式 | 说明 |
|----------|------|------|
| 灵感 | `.md` | 只输出最终版 |
| 大纲（全书） | `.md` | 最终版 + 角色系统 |
| 章节大纲 | `.md` | 最终版 + 世界线系统 + 角色剧情 |
| 正文 | `.txt` | 纯正文，不含任何评价、优化、注释内容 |
| 角色系统 | `.md` | 角色档案 |
| 世界线系统 | `.md` | 世界线进程追踪 |
| 设定 | `.md` | 世界观文档 |

## 角色系统模板

每个小说维护一个角色系统文件 `角色系统_[小说名].md`，记录所有角色的状态追踪。

```markdown
# 角色系统 — [小说名]

## 角色列表

### [角色名]

**核心特点**：[3词定义，如：高冷/外冷内热/护短]

**当前状态**：
- 身份：[当前身份]
- 能力：[当前能力水平]
- 情感状态：[对谁/什么感觉]

**关系网络**：
- [角色B]：[关系描述] → [变化趋势]
- [角色C]：[关系描述] → [变化趋势]

**成长弧线**：
- 起点：[初始状态]
- 转折1：[第X章，发生了什么，导致什么变化]
- 转折2：[...]
- 当前：[现在处于弧线的什么位置]

**故事线进展**：
- [第X章]：[关键事件]
- [第X章]：[关键事件]
```
