# 检查系统规则集

## 五维度一致性检查

### 1. 角色一致性

检查项：
- 角色名是否前后一致
- 角色性格是否符合设定
- 角色语言风格是否一致（同一角色在不同场景可有不同情绪×行为×语言组合，但核心个性一致）
- 角色关系是否符合前文发展
- 角色外貌描述是否矛盾

### 2. 时间线一致性

检查项：
- 事件时间顺序是否合理
- 角色年龄是否对得上
- 时间跨度是否合理
- 是否有时间悖论

### 3. 世界观一致性

检查项：
- 设定规则是否被违反
- 力量体系是否前后一致
- 地理位置是否矛盾
- 势力关系是否符合设定

### 4. 伏笔追踪

检查项：
- 已埋伏笔是否被遗忘
- 伏笔回收是否自然
- 是否有未解释的悬念
- 伏笔密度是否合理

### 5. 关系进展

检查项：
- 角色关系变化是否有铺垫
- 感情线发展是否自然
- 亲密程度是否符合节奏
- 是否有关系跳跃

## 去耦合一致性检查

### 角色维度多样性

检查同一角色在不同场景是否展现了不同的情绪×行为×语言组合：

| 检查项 | 标准 | 问题示例 |
|--------|------|----------|
| 情绪变化 | 同一角色至少展现3种不同情绪 | "高冷型"角色永远高冷 |
| 行为变化 | 同一角色至少有2种不同行为 | "毒舌"角色只会毒舌 |
| 语言变化 | 同一角色至少有2种语言风格 | 所有场景都"惜字如金" |

### 情节元素独立性

检查事件类型、事件风格、场景氛围是否独立选择：

| 检查项 | 标准 | 问题示例 |
|--------|------|----------|
| 事件类型 | 不重复使用同一事件类型 | 连续3章都是"误会" |
| 事件风格 | 不重复使用同一事件风格 | 所有冲突都是"搞笑风格" |
| 场景氛围 | 不重复使用同一场景氛围 | 所有亲密都在"私密氛围" |

## 写作反馈生成

每次检查后，自动生成 `追踪/写作反馈.md`，包含：

### 高频问题清单

列出本次检查发现的高频问题，供后续章节规避。

### 角色注意事项

列出已发现的角色问题和写作建议。

### 伏笔预警

列出即将超期/已超期的伏笔。

### 世界观约束提醒

列出易错的世界观设定。

### 上一章衔接要点

列出上一章的结束状态和待处理悬念。

## 自动管理机制

### 触发条件

- 正文/文件被修改时自动触发
- 设定/文件被修改时自动触发
- 每批次完成后自动触发

### 防抖机制

- 同一文件在 60 秒内的多次修改，只触发一次检查
- 锁文件：`.claude/.check-lock`
- 检查完成后自动清除

### 反馈循环

```
正文生成 → 自动检查 → 写作反馈 → 下一章读取反馈 → 正文生成
    ↑                                              ↓
    └──────────────────────────────────────────────┘
```
