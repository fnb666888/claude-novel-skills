# 检查系统规则集

## 五维度一致性检查

### 1. 角色一致性

检查项：
- 角色名是否前后一致
- 角色性格是否符合设定
- 角色语言风格是否一致（同一角色在不同场景可有不同情绪×行为×语言组合，但核心个性一致）
- 角色关系是否符合前文发展
- 角色外貌描述是否矛盾

### 2. 时间线一致性

检查项：
- 事件时间顺序是否合理
- 角色年龄是否对得上
- 时间跨度是否合理
- 是否有时间悖论

### 3. 世界观一致性

检查项：
- 设定规则是否被违反
- 力量体系是否前后一致
- 地理位置是否矛盾
- 势力关系是否符合设定

### 4. 伏笔追踪

检查项：
- 已埋伏笔是否被遗忘
- 伏笔回收是否自然
- 是否有未解释的悬念
- 伏笔密度是否合理

### 5. 关系进展

检查项：
- 角色关系变化是否有铺垫
- 感情线发展是否自然
- 亲密程度是否符合节奏
- 是否有关系跳跃

## 去耦合一致性检查

### 角色维度多样性

检查同一角色在不同场景是否展现了不同的情绪×行为×语言组合：

| 检查项 | 标准 | 问题示例 |
|--------|------|----------|
| 情绪变化 | 同一角色至少展现3种不同情绪 | "高冷型"角色永远高冷 |
| 行为变化 | 同一角色至少有2种不同行为 | "毒舌"角色只会毒舌 |
| 语言变化 | 同一角色至少有2种语言风格 | 所有场景都"惜字如金" |

### 情节元素独立性

检查事件类型、事件风格、场景氛围是否独立选择：

| 检查项 | 标准 | 问题示例 |
|--------|------|----------|
| 事件类型 | 不重复使用同一事件类型 | 连续3章都是"误会" |
| 事件风格 | 不重复使用同一事件风格 | 所有冲突都是"搞笑风格" |
| 场景氛围 | 不重复使用同一场景氛围 | 所有亲密都在"私密氛围" |

## 写作反馈生成

每次检查后，自动生成 `追踪/写作反馈.md`，包含：

### 高频问题清单

列出本次检查发现的高频问题，供后续章节规避。

### 角色注意事项

列出已发现的角色问题和写作建议。

### 伏笔预警

列出即将超期/已超期的伏笔。

### 世界观约束提醒

列出易错的世界观设定。

### 上一章衔接要点

列出上一章的结束状态和待处理悬念。

## 上下文摘要生成

每次检查后，同步更新 `追踪/上下文摘要.md`，用于新会话/子Agent快速恢复上下文。

### 即时检查时更新

- 当前进度（已完成章节号）
- 上章衔接要点（结束场景、待处理悬念、情绪基调）

### 批次检查时完整更新

- 当前进度
- 核心角色状态（每个角色一句话概括当前状态）
- 活跃伏笔表（伏笔、埋设章节、状态、下一步）
- 世界观关键约束（最重要的3-5条规则）
- 写作风格要求（核心体验类型、极端化方向、去AI味重点）
- 上章衔接要点

### 设计原则

上下文摘要的目标是：**一个新会话读完这一个文件，就能立刻开始续写，不会跑偏**。
内容必须精炼、结构化，避免冗长叙述。

## 分层检查机制

### 即时检查（每章完成时触发）

轻量级检查，确保基本一致性，不打断写作流。

**检查内容**：
- 角色名是否与角色系统一致
- 时间线是否与世界线系统一致
- 本章提到的伏笔是否在伏笔状态表中
- 角色当前状态是否与角色系统一致
- 上章衔接点是否正确处理

**输出**：
- 更新 `~/小说/CLAUDE.md` 的"当前状态"部分（进度、角色状态、上章衔接点）
- 更新 `追踪/上下文摘要.md` 的"当前进度"和"上章衔接点"部分
- 如发现问题，记录到 `追踪/写作反馈.md` 的"高频问题清单"

### 批次检查（每批10章完成后触发）

完整检查，覆盖五维度 + 去耦合一致性。

**检查内容**：
- 完整五维度一致性检查（角色、时间线、世界观、伏笔、关系）
- 去耦合一致性检查（角色多样性、情节独立性）
- 高频问题统计与趋势分析
- 伏笔预警生成（即将超期/已超期）
- 世界观约束提醒

**输出**：
- 完整更新 `~/小说/CLAUDE.md` 所有部分（含新发现的写作规则问题）
- 完整更新 `追踪/写作反馈.md` 所有部分
- 完整更新 `追踪/上下文摘要.md` 所有部分

### 全局检查（全书完成后触发）

深度检查，覆盖全书范围的一致性。

**检查内容**：
- 全书角色弧线完整性（每个角色的成长线是否闭合）
- 所有伏笔是否闭合（无遗漏的伏笔）
- 世界观一致性全文扫描（跨全书的设定矛盾）
- 关系进展全链路检查（感情线是否有跳跃）
- 时间线全文验证（无悖论）

**输出**：
- `追踪/全局检查报告.md` — 全书一致性报告
- 最终更新 `~/小说/CLAUDE.md`（标记全书完成）
- 最终更新 `追踪/上下文摘要.md`（全书完成版）

### 反馈循环

```
正文生成 → 即时检查 → 更新CLAUDE.md+上下文摘要 → 下一章自动加载CLAUDE.md → 正文生成
    ↓（每批完成）
批次检查 → 更新CLAUDE.md+写作反馈+上下文摘要 → 下一批自动加载CLAUDE.md → 正文生成
    ↓（全书完成）
全局检查 → 全局检查报告 → 最终修订
```
