# 对比优化规则集

## 核心哲学

**不靠死板的评分标准，而是通过与优秀作品的对比来提升质量。**

- 将生成内容与 `references/` 目录下的优秀小说进行对比
- 从结构、内容、技法等多个维度找出差距
- 针对性优化，使输出达到参考作品的水平

## 参考资料管理

### 参考小说位置

`references/` 目录下存放用户认可的优秀小说文件（支持 `.txt` 和 `.md` 格式）。

### 读取策略

1. 扫描 `references/` 目录下的所有小说文件
2. 根据当前创作类型，选择同类型作品作为主要参考
3. 分析参考作品的特点和优势，建立对比基准

## 对比维度

### 全面对比框架

| 维度 | 对比内容 | 优化方向 |
|------|----------|----------|
| **结构设计** | 叙事结构、章节安排、伏笔设置、节奏控制 | 对齐参考作品的结构水平 |
| **内容描写** | 场景描写、感官细节、氛围营造、情感渲染 | 达到参考作品的描写细腻度 |
| **人物塑造** | 角色设计、对话写作、行为刻画、性格展现 | 参考作品的人物立体感 |
| **写作技法** | 钩子设置、悬念控制、情感节奏、高潮设计 | 学习参考作品的技法运用 |
| **节奏控制** | 快慢交替、张弛有度、章节节奏、情绪节奏 | 匹配参考作品的节奏感 |
| **对话质量** | 口语化程度、潜台词、人设化、信息效率 | 参考作品的对话水准 |
| **情感渲染** | 情感浓度、渲染手法、读者共情、情绪波动 | 达到参考作品的情感强度 |

### 灵感对比重点

对比参考作品的：
- 核心设定的吸引力
- 角色设计的反差魅力
- 情节展开的可能性
- 日常场景的可塑性

### 大纲对比重点

对比参考作品的：
- 整体结构设计
- 章节节奏安排
- 伏笔埋设与回收
- 高潮节点设置
- 人物成长弧线

### 正文对比重点

对比参考作品的：
- 场景描写的细腻度
- 对话的自然度和信息量
- 情感渲染的浓度
- 节奏控制的精准度
- 金句的记忆点

## 对比优化流程

### 第一步：参考分析

1. 读取 `references/` 目录下的小说文件
2. 分析参考作品的核心优势：
   - 结构上有什么特点？
   - 描写上有什么亮点？
   - 技法上有什么值得学习的？
3. 建立本次创作的对比基准

### 第二步：内容对比

将生成的内容与参考作品进行逐维度对比：

1. **结构层面**：我的结构设计是否达到参考水平？
2. **内容层面**：我的描写细腻度是否匹配参考？
3. **技法层面**：我是否学到了参考作品的技法？
4. **节奏层面**：我的节奏控制是否到位？
5. **情感层面**：我的情感渲染是否足够？

### 第三步：差距识别

明确指出：
- 哪些方面已经达到或超过参考水平
- 哪些方面还有差距
- 差距的具体表现是什么

### 第四步：针对性优化

根据差距识别的结果：
- 保留已经达到参考水平的部分
- 针对差距部分进行定向优化
- 学习参考作品的技法，应用到当前内容

### 第五步：效果验证

优化后再次与参考作品对比：
- 差距是否缩小？
- 是否达到参考水平？
- 是否保持了原有内容的核心魅力？

## 优化频率

### 每章：即时检查（轻量）

每章完成后只做硬性一致性检查（角色名、时间线、伏笔衔接），不做完整优化。
目的是确保基本连贯性，不打断写作流。

### 每弧（故事弧完成后）：完整对比优化

每个故事弧完成后执行完整的七维度对比优化 + 去AI味处理。故事弧章数不固定（3-50章），根据自然叙事节点结束。

**原因**：
1. 故事弧是自然的优化单元——有完整的起承转合，优化时有足够上下文
2. 跨章节能看到角色弧线是否完整、伏笔是否自然、节奏是否合理
3. token效率更高——每次优化需读取参考小说+多章正文+追踪文件，频繁做成本高

**注意**：如果故事弧超过15章，建议在弧内分段优化（每10-15章一次），避免单次优化上下文过大。

**执行方式**：
1. 读取本批所有章节
2. 读取 `references/` 下的参考小说（采样，不必全读）
3. 七维度全面对比
4. 去AI味处理（`rules/deslop-rules.md` 的6 Gate + 三遍法）
5. 更新追踪文件
6. 输出优化后的版本

### 全书完成：全局优化

全书完成后做一次最终的全局优化，确保跨全书的质量一致性。

## 极端化原则

**好小说不是各维度都高分，而是某个维度做到极致。**

对比优化时注意：
- 暧昧情节就要足够暧昧——让人心跳加速到脸红
- 性爱情节就要足够诱惑——让人血脉贲张
- 悲伤情节就要足够悲伤——让人泪流不止
- 搞笑情节就要足够搞笑——让人笑出声
- 悬疑情节就要足够悬疑——让人猜不到结局
- 热血情节就要足够热血——让人热血沸腾

优化时优先增强内容的自然优势维度，而非平均用力。

## 去耦合化评估

**好小说的角色和情节不是绑定的，而是自由组合的。**

对比时检查：
1. **角色维度多样性**：同一角色在不同场景是否展现了不同的情绪×行为×语言组合？
2. **情节元素独立性**：事件类型、事件风格、场景氛围是否独立选择？
3. **交叉组合新鲜感**：角色特点与情节元素的组合是否给读者带来意外？

## 达标标准

以参考作品为基准：
- 达到参考作品的核心水平 → 达标
- 超过参考作品 → 优秀
- 未达到参考作品 → 继续优化

如果 `references/` 目录为空，则使用以下默认标准：
- 结构完整、逻辑自洽
- 描写细腻、有画面感
- 对话自然、有人设感
- 节奏合理、有起伏感
- 情感到位、有共鸣感
