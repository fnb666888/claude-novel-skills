# 小说灵感迭代生成系统

**核心理念**：灵感不是一次生成的，而是通过"头脑风暴→资料查询→生成→评分→优化"的迭代循环打磨出来的。

**本skill整合全部创作工具**，自动完成全流程。

---

## 整合工具清单

| 工具 | 整合阶段 | 作用 |
|:---|:---|:---|
| **novel-scan** | 阶段零：市场调研（可选） | 数据驱动的选题方向约束 |
| **内部头脑风暴** | 阶段一：创意萌发 | 发散思考，生成3-5个创意方向 |
| **WebSearch** | 阶段二：资料支撑 | 搜索背景资料、历史事件、职业设定 |
| **novel-worldbuilding** | 阶段二+六 | 创建/读取世界观设定文档 |
| **novel-writing** | 阶段三：灵感生成 | 提供角色类型、圣体系统、情节模板 |
| **novel-evaluation** | 阶段四：吸引力评分 | 6维度评分体系（量表A） |
| **novel-optimization** | 阶段五：定向优化 | 针对短板的优化技法 |

> **说明**：本工具已移除伪整合（brainstorming/context7/notepad/craft/brain-methodology），改为内部实现或真实工具调用。

## 使用方式

### 基础用法
- `/novel-inspiration` — 完整流程（头脑风暴→生成→迭代优化）
- `/novel-inspiration [主题]` — 围绕指定主题生成

### 指定类型
- `/novel-inspiration 都市重生` — 指定类型
- `/novel-inspiration 玄幻修仙` — 指定类型
- `/novel-inspiration 校园青春` — 指定类型

### 流程控制
- `/novel-inspiration --max 3` — 最多迭代3轮（默认10轮）
- `/novel-inspiration --target 8.5` — 目标分数8.5（默认9.0）
- `/novel-inspiration --no-brainstorm` — 跳过头脑风暴，直接生成
- `/novel-inspiration --no-research` — 跳过资料查询

### 达标标准
- **默认目标分数：9.0**（极品灵感）
- **达标判定：总分 ≥ 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

## 阶段一：创意萌发（brainstorming）

**工具**：brainstorming（头脑风暴）

**目标**：在正式生成灵感前，先进行头脑风暴，探索多个创意方向。

### 执行流程

```
1. 根据用户指定的主题（如有），进行发散思考
2. 生成3-5个不同的创意方向，每个方向包含：
   - 核心概念（一句话）
   - 类型定位（都市/玄幻/校园等）
   - 独特卖点（为什么读者会看）
   - 潜在风险（可能的问题）
3. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 创意头脑风暴

## 主题：[用户指定主题 或 自动选择]

## 创意方向

### 方向A：[名称]
- 核心概念：[一句话]
- 类型定位：[类型]
- 独特卖点：[最吸引人的点]
- 潜在风险：[可能的问题]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```

### 执行流程

```

   - 职业设定（如需要）
   - 地理环境（如需要）
   - 文化元素（如需要）
   - 科学知识（如需要）
3. 读取novel-worldbuilding中的已有设定（如有）
4. 记录关键资料点，用于后续灵感生成
```

## 阶段三：灵感生成（novel-writing）

**工具**：novel-writing（创作技法工具箱）

**目标**：基于头脑风暴和资料支撑，生成完整的150万字容量灵感。

### 执行流程

```
1. 读取novel-writing中的角色类型库（模块A）
2. 随机抽取角色类型，确保多样性
3. 从反差感塑造法中获取反差点公式
4. 从圣体系统中随机组合圣体属性
5. 从语言风格差异化表中匹配语言风格
6. 设计女主矩阵（8-12位）
7. 设计反派矩阵（4级）
8. 设计多卷架构（4-6卷）
9. 设计支线体系（8条以上）
10. 进行150万字容量验证
```

### 150万字容量标准

| 维度 | 最低要求 | 说明 |
|------|----------|------|
| **女主数量** | 8-12位 | 每位女主独立感情线可写10-15万字 |
| **反派层级** | 至少4级 | 小BOSS→中BOSS→大BOSS→终极BOSS |
| **世界观深度** | 至少3层空间/势力 | 随剧情推进不断解锁新地图/势力 |
| **支线数量** | 至少8条独立支线 | 每条支线可独立展开5-10万字 |
| **卷数规划** | 4-6卷 | 每卷25-40万字，有独立高潮和阶段性结局 |

## 阶段四：吸引力评分（novel-evaluation）

**工具**：novel-evaluation（评价体系）

**目标**：量化评估灵感的吸引力，识别短板。

### 评分维度（量表A）

| 维度 | 权重 | 核心问题 | 评分标准 |
|------|------|----------|----------|
| **亲密潜力** | 30% | 设定能产生多少高张力亲密场景？ | 9-10:天然禁忌感 7-8:有空间 5-6:需设计 3-4:关联弱 1-2:无可能 |
| **反差魅力** | 25% | 角色是否有"没想到"的惊喜感？ | 9-10:强烈反差 7-8:有效果 5-6:一般 3-4:弱 1-2:扁平 |
| **趣味基因** | 20% | 设定本身是否有趣/好笑？ | 9-10:自带喜感 7-8:有潜力 5-6:一般 3-4:严肃 1-2:无趣 |
| **意外系数** | 15% | 有多少出人意料的可能性？ | 9-10:多个反转 7-8:有惊喜 5-6:少 3-4:中规中矩 1-2:套路 |
| **日常可塑性** | 5% | 能否写出有烟火气的日常？ | 9-10:天然场景 7-8:有空间 5-6:需设计 3-4:脱节 1-2:无场景 |
| **上瘾钩子** | 5% | 核心设定是否自带驱动力？ | 9-10:强驱动 7-8:有悬念 5-6:一般 3-4:弱 1-2:无钩子 |

### 评分计算

```
总分 = 亲密潜力×30% + 反差魅力×25% + 趣味基因×20% + 意外系数×15% + 日常可塑性×5% + 上瘾钩子×5%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **✓ 采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的灵感才会被采用并保存为最终文件。**

## 阶段五：定向优化（novel-optimization）

**工具**：novel-optimization（优化策略）

**目标**：针对评分短板进行定向优化，补充新的钩子元素。

### 优化策略（根据短板选择）

#### 亲密潜力优化（权重30%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 缺乏禁忌感 | 身份禁忌极致化 | 模块C | 设计"绝对不能碰"的禁忌线 |
| 缺乏身份差 | 反差感塑造法 | 模块A | 设计身份反差（师姐/女帝/仙子） |
| 缺乏亲密场景 | 渐进式推进法 | 模块B | 设计四阶段亲密递进 |
| 缺乏张力 | 克制留白法 | 模块C | 设计"差一点"的克制场景 |

#### 反差魅力优化（权重25%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 角色扁平 | 核心六要素 | 模块A | 添加反差感、圣体、语言风格 |
| 缺乏惊喜 | 表里双层叙事法 | 模块A | 设计表面vs内心的双层 |
| 缺乏记忆点 | 一句话定义法 | 模块A | 用一句话标签定义角色 |
| 缺乏辨识度 | 行为标签化 | 模块A | 设计标志性行为/物品 |

#### 趣味基因优化（权重20%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 设定严肃 | 一句话定义法 | 模块A | 找到设定的喜感角度 |
| 缺乏笑点 | 幽默技巧 | 模块C | 添加内心独白式幽默 |
| 缺乏反差 | 对比式幽默 | 模块C | 设计极端并置的喜剧效果 |
| 缺乏日常 | 场景构建 | 模块C | 设计有烟火气的日常场景 |

#### 意外系数优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 设定套路 | 圣体系统 | 模块A | 添加独特的圣体属性 |
| 缺乏反转 | 信息差驱动法 | 模块B | 设计读者vs角色的信息差 |
| 缺乏悬念 | 嵌套式叙事框架 | 模块B | 设计"故事中的故事" |
| 缺乏深度 | 身份反转法 | 模块A | 设计隐藏身份/命运反差 |

### craft钩子补充

**工具**：craft（钩子发现）

在优化过程中，使用craft发现新的钩子和吸引力元素：
- 发现新的亲密场景钩子
- 发现新的幽默触发点
- 发现新的悬念设置方式
- 发现新的角色魅力点

### 优化流程

```
1. 分析当前灵感的短板（得分<7分的维度）
2. 从novel-writing中选择对应的优化技法
3. 使用craft发现新的钩子元素
4. 保留核心设定，针对性增强薄弱环节
5. 输出优化版灵感
6. 重新评分，判断是否达标
7. 未达标则继续优化循环
```

## 迭代控制逻辑

### 迭代判定规则

```python
# 伪代码逻辑（实际执行时由Claude内部实现）
target_score = 9.0  # 默认目标分数
max_rounds = 10     # 默认最大轮数
current_round = 0
best_score = 0
best_inspiration = None
versions = []       # 版本回溯：保留每轮内容

# 阶段一：头脑风暴（内部发散思考）
# 生成3-5个创意方向，每个含：核心概念/类型/卖点/风险
selected_direction = user_select_or_auto_best(创意方向列表)

# 阶段二：资料支撑
# 使用WebSearch搜索背景资料
research_data = web_search(selected_direction.需要查询的资料)
# 读取已有世界观设定（如有）
worldbuilding = read_worldbuilding_if_exists()

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        # 阶段三：初始生成（使用novel-writing技法）
        inspiration = generate_inspiration(selected_direction, research_data)
    else:
        # 阶段五：定向优化（使用novel-optimization技法）
        inspiration = optimize_inspiration(inspiration, weaknesses)

    # 阶段四：评分（使用novel-evaluation量表A）
    score_report = evaluate_inspiration(inspiration)
    current_score = score_report.total_score

    # 版本追踪：保留每轮内容
    versions.append({
        round: current_round,
        score: current_score,
        content: inspiration,
        weaknesses: score_report.weaknesses
    })

    # 记录最优版本
    if current_score > best_score:
        best_score = current_score
        best_inspiration = inspiration

    # 判断是否达标
    if current_score >= target_score:
        # 阶段六：深度分析 + 世界观初始化
        deep_analysis = analyze_inspiration(inspiration)
        create_worldbuilding_draft(inspiration)  # 创建世界观初稿
        output_final_inspiration(inspiration, score_report, deep_analysis, versions)
        break

    # 输出当轮报告
    output_round_report(inspiration, score_report, current_round)

# 达到最大轮数，输出最优版本（标注未达标）
if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_inspiration(best_inspiration)
    create_worldbuilding_draft(best_inspiration)
    output_best_inspiration(best_inspiration, best_score, versions, note="未达标")
```

## 输出文档规则

### 重要：只保留最终达标文件

**本skill只保存最终达标的灵感文件（9.0+分），中间迭代过程不保存为文件。**

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：`inspiration_[主题]_[时间戳].md`

### 保存条件
- **只有总分 ≥ 9.0 的灵感才会保存为文件**
- 未达标的迭代过程仅在对话中显示，不保存文件
- 达到最大轮数仍未达标时，保存当前最优版本（需标注"未达标"）

### 输出内容
最终文件包含：
1. 完整灵感卡（达标版）
2. 最终评分报告
3. 深度分析报告
4. 迭代历程总结（每轮分数变化，仅文字记录）

### 灵感卡标准格式（供下游工具解析）
