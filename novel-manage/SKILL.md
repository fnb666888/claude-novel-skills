# 小说项目管理工具

**核心理念**：世界观是小说的"宪法"，一致性是读者信任的基石，伏笔是叙事的暗线，进度是创作的节奏。统一管理，才能驾驭百万字长篇。

**本工具合并原 novel-worldbuilding 与 novel-consistency**，并新增伏笔管理、进度追踪、项目维护功能。

---

## 子命令路由总览

| 子命令 | 功能 | 来源 |
|--------|------|------|
| `世界观` | 世界观管理（新建/查看/编辑/检查） | 合并自 novel-worldbuilding |
| `检查` | 一致性检查（全书/单章/专项） | 合并自 novel-consistency |
| `伏笔` | 伏笔管理（新增/查看/回收/扫描） | 新增 |
| `进度` | 进度追踪（查看/更新） | 新增 |
| `结构` | 项目目录验证 | 新增 |
| `备份` | 项目状态打包 | 新增 |
| `迁移` | 目录结构升级 | 新增 |

---

## 使用方式

### 世界观管理
- `/novel-manage 世界观 新建 [小说名]` — 从零创建世界观设定文档
- `/novel-manage 世界观 查看 [小说名]` — 查看当前世界观设定
- `/novel-manage 世界观 编辑 [小说名]` — 编辑世界观设定
- `/novel-manage 世界观 检查 [小说名] [待检查文件]` — 检查文件是否符合世界观
- `/novel-manage 世界观 从灵感 [灵感文件路径]` — 从已有灵感提取世界观
- `/novel-manage 世界观 从大纲 [大纲文件路径]` — 从已有大纲提取世界观

### 一致性检查
- `/novel-manage 检查 [小说名]` — 全书五维度一致性检查
- `/novel-manage 检查 [小说名] --from 第X章 --to 第Y章` — 检查指定范围
- `/novel-manage 检查章 [章节文件路径]` — 单章与前文连续性检查
- `/novel-manage 检查章 [章节文件路径] --previous [上一章文件路径]` — 指定上一章
- `/novel-manage 角色 [小说名]` — 只检查角色一致性
- `/novel-manage 伏笔检查 [小说名]` — 只检查伏笔回收
- `/novel-manage 时间线 [小说名]` — 只检查时间线
- `/novel-manage 世界观检查 [小说名]` — 只检查世界观一致性

### 伏笔管理
- `/novel-manage 伏笔 新增 [小说名]` — 记录新伏笔
- `/novel-manage 伏笔 查看 [小说名]` — 查看所有伏笔状态
- `/novel-manage 伏笔 回收 [小说名] [伏笔ID]` — 标记伏笔回收
- `/novel-manage 伏笔 扫描 [小说名]` — 扫描未回收伏笔

### 进度追踪
- `/novel-manage 进度 查看 [小说名]` — 查看当前创作进度
- `/novel-manage 进度 更新 [小说名]` — 更新进度记录

### 项目维护
- `/novel-manage 结构 验证 [小说名]` — 检查目录完整性
- `/novel-manage 备份 [小说名]` — 打包当前项目状态
- `/novel-manage 迁移 [小说名]` — 目录结构升级

# 一、世界观管理

**核心理念**：世界观是小说的"宪法"，所有创作工具必须在世界观框架内运行，不能自相矛盾。

---

## 1.1 创建世界观

### 新建
```
/novel-manage 世界观 新建 [小说名]
```

执行流程：
1. 创建标准目录结构（如不存在）
2. 根据小说类型生成世界观模板
3. 填入基础框架信息
4. 保存至 `设定/世界观.md`

### 从灵感提取
```
/novel-manage 世界观 从灵感 [灵感文件路径]
```

执行流程：
1. 读取灵感文件
2. 提取世界观相关要素（世界类型、力量体系、势力、地理等）
3. 填入8节标准格式模板
4. 保存至对应小说的 `设定/世界观.md`

### 从大纲提取
```
/novel-manage 世界观 从大纲 [大纲文件路径]
```

执行流程：
1. 读取大纲文件
2. 提取世界观相关要素
3. 扩展为完整的8节标准格式
4. 保存至对应小说的 `设定/世界观.md`

---

## 1.2 查看与编辑

### 查看
```
/novel-manage 世界观 查看 [小说名]
```

输出当前世界观设定的完整内容，按8节结构分段展示。

### 编辑
```
/novel-manage 世界观 编辑 [小说名]
```

进入编辑模式，支持：
- 编辑单个章节（如"编辑 第三节 势力分布"）
- 添加新条目（如"添加角色 林风眠"）
- 更新状态（如"更新角色状态 林风眠 当前：筑基期"）

### 检查
```
/novel-manage 世界观 检查 [小说名] [待检查文件]
```

检查指定文件是否符合世界观设定，输出一致性检查报告（详见一致性检查章节）。

# 二、一致性检查

**核心理念**：读者最怕的不是写得不好，而是前后矛盾。一致性检查帮你避免"硬伤"。

---

## 2.1 检查维度（五维度）

### 维度一：角色一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 性格一致性 | 角色行为是否符合已建立的性格 | 硬伤 |
| 语言风格一致性 | 对话风格是否前后一致 | 中等 |
| 能力一致性 | 角色能力是否超出设定 | 硬伤 |
| 外貌一致性 | 外貌描述是否前后矛盾 | 中等 |
| 称呼一致性 | 角色间称呼是否正确 | 中等 |
| 状态一致性 | 角色当前状态（位置/伤势/情绪）是否正确 | 硬伤 |

#### 检查方法
```
1. 从世界观文档第七节读取角色档案
2. 扫描全书，提取每个角色的行为/对话/状态
3. 对比角色档案，发现不一致之处
4. 输出检查报告
```

#### 检查示例
```
问题：第3章描写林风眠"修为刚入炼气期"，但第15章他"轻松击败筑基期敌人"
→ 硬伤：修为设定矛盾
→ 建议：修改第15章，增加合理的解释（如使用金手指/法宝）
```

### 维度二：伏笔回收

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 未回收伏笔 | 已埋设但未回收的伏笔 | 硬伤 |
| 回收矛盾 | 回收方式与埋设时的暗示不符 | 硬伤 |
| 重复伏笔 | 同一个伏笔被埋设多次 | 中等 |
| 超期伏笔 | 超过预计回收章节20章以上未回收 | 中等 |

#### 检查方法
```
1. 从世界观文档第八节和追踪/伏笔.md读取伏笔追踪表
2. 扫描全书，标记伏笔的埋设和回收
3. 检查：
   - 埋设的伏笔是否都有回收
   - 回收方式是否与暗示一致
   - 是否有重复埋设
   - 是否有超期未回收
4. 输出检查报告
```

#### 检查示例
```
问题：第5章提到"双鱼佩在关键时刻会发光"，但全书再未提及
→ 未回收伏笔
→ 建议：在后续章节中回收（如第50章用双鱼佩打断亲密场景）
```

### 维度三：时间线一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 时间顺序矛盾 | 事件发生的先后顺序不合理 | 硬伤 |
| 时间跨度矛盾 | 两事件之间的时间间隔不合理 | 中等 |
| 年龄矛盾 | 角色年龄与时间流逝不符 | 硬伤 |
| 季节矛盾 | 季节/天气描述与时间不符 | 中等 |

#### 检查方法
```
1. 从世界观文档第五节和追踪/时间线.md建立时间线
2. 标记每个事件的发生时间
3. 检查时间顺序和间隔
4. 输出检查报告
```

#### 检查示例
```
问题：第10章是"春天"，第12章是"三天后"却描写"漫天飞雪"
→ 季节矛盾
→ 建议：修改天气描述或时间间隔
```

### 维度四：关系状态一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 关系进展 | 关系发展是否符合前文铺垫 | 硬伤 |
| 称呼变化 | 称呼是否随关系变化而变化 | 中等 |
| 情感状态 | 角色情感状态是否正确 | 硬伤 |
| 多角关系 | 多条感情线是否冲突 | 中等 |

#### 检查方法
```
1. 从世界观文档第七节读取角色关系表
2. 扫描全书，标记关系变化节点
3. 检查关系进展是否合理
4. 输出检查报告
```

### 维度五：世界观一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 等级体系 | 等级描述是否前后一致 | 硬伤 |
| 势力关系 | 势力间关系是否前后一致 | 硬伤 |
| 地理距离 | 地点间距离/方位是否矛盾 | 硬伤 |
| 规则违反 | 是否违反已建立的世界规则 | 硬伤 |
| 物品追踪 | 关键物品的去向是否明确 | 中等 |
| 能力限制 | 角色能力是否超出设定限制 | 中等 |

#### 检查方法
```
1. 从世界观文档第一至六节读取所有设定
2. 扫描全书，提取世界观相关名词（地名、势力名、等级名、能力名）
3. 逐项对比，发现矛盾
4. 输出检查报告
```

## 2.2 严重程度定义

| 级别 | 名称 | 定义 | 处理方式 |
|------|------|------|----------|
| **硬伤** | 必须修复 | 直接矛盾、逻辑不通、违反已建立规则 | 必须修复后才能继续生成，提供具体修改建议 |
| **中等** | 建议修复 | 不够合理、可能引起读者困惑 | 建议修复，可在后续章节自然修正 |
| **轻微** | 可选修复 | 细节瑕疵、不影响大局 | 记录在案，供用户参考 |

## 2.5 写作反馈生成

**核心理念**：检查不是终点，反馈才是起点。一致性检查的结果必须转化为可执行的写作指导，让后续章节避免重复犯错。

`追踪/写作反馈.md`

### 反馈生成时机

|----------|------|

```markdown
- 检查类型：[全书检查/单章检查/批次检查]

| # | 问题类型 | 具体表现 | 出现次数 | 涉及章节 | 写作指导 |
|---|----------|----------|----------|----------|----------|
| 1 | 角色语言风格不一致 | 林风眠有时文言有时白话 | 3次 | 第5、8、12章 | 统一使用白话+偶尔文言点缀 |

|--------|----------|----------|------|------|

```

### 反馈生成规则

**高频问题提取**：
1. 汇总检查报告中的所有"硬伤"和"中等问题"
2. 按问题类型分类（角色一致性/时间线/世界观/伏笔/关系）
3. 统计每类问题的出现次数
4. 针对每个问题给出具体的"写作指导"（不是修复建议，而是预防建议）

**角色注意事项**：
1. 从检查报告中提取每个角色的问题
2. 转化为"写作时应该怎么做"的建议
3. 如角色无问题，不列入

**伏笔预警**：
1. 从伏笔追踪中提取即将超期（距预计回收<5章）和已超期的伏笔
2. 给出回收建议

**世界观约束提醒**：
1. 从世界观一致性检查中提取易错点
2. 转化为写作时的约束条件

**上一章衔接要点**：
1. 从最新章节的状态中提取结束状态
2. 从章末钩子中提取待处理悬念
3. 从伏笔和事件记录中提取需要延续的线索

## 2.6 修复建议规则

### 硬伤修复
- 必须修复后才能继续生成
- 提供具体的修改建议
- 如果矛盾严重，建议重新生成该章节
- 修复后需重新检查确认

### 中等问题修复
- 建议修复，但不强制
- 提供修改方向
- 可以在后续章节中自然修正

### 轻微问题修复
- 可选修复
- 记录在案，供用户参考

# 三、伏笔管理

**核心理念**：伏笔是叙事的暗线，好的伏笔让读者回味无穷，遗漏的伏笔让读者失望。系统化管理，确保每个伏笔都有归宿。

---

## 3.1 伏笔操作

### 新增伏笔
```
/novel-manage 伏笔 新增 [小说名]
```

执行流程：
1. 提示用户输入伏笔信息（或从当前章节自动提取）
2. 分配唯一ID（F001, F002, ...）
3. 记录埋设位置、计划回收位置、关联角色
4. 更新 `追踪/伏笔.md` 和世界观文档第八节

### 查看伏笔
```
/novel-manage 伏笔 查看 [小说名]
```

输出所有伏笔的状态，支持筛选：
- `/novel-manage 伏笔 查看 [小说名] --status 未回收` — 只看未回收的
- `/novel-manage 伏笔 查看 [小说名] --status 已回收` — 只看已回收的
- `/novel-manage 伏笔 查看 [小说名] --overdue` — 只看超期未回收的
- `/novel-manage 伏笔 查看 [小说名] --character [角色名]` — 只看关联某角色的

### 回收伏笔
```
/novel-manage 伏笔 回收 [小说名] [伏笔ID]
```

执行流程：
1. 确认伏笔存在且未回收
2. 记录回收章节和回收方式
3. 更新伏笔状态为"已回收"
4. 更新 `追踪/伏笔.md` 和世界观文档第八节

### 扫描未回收伏笔
```
/novel-manage 伏笔 扫描 [小说名]
```

执行流程：
1. 读取所有未回收伏笔
2. 扫描最新章节，检查是否有隐性回收
3. 标记超期伏笔（超过预计回收章节20章以上）
4. 输出扫描报告

## 3.4 伏笔管理规则

### 埋设规则
- 每个伏笔必须有唯一ID
- 必须记录埋设位置（章节+段落）
- 必须规划回收章节（不能无限期搁置）
- 必须标注关联角色

### 回收规则
- 回收方式必须与埋设时的暗示一致
- 回收后需更新所有相关文档
- 回收后需检查是否触发新的伏笔

### 超期处理
- 超过预计回收章节20章以上标记为"超期"
- 超期伏笔需优先处理
- 可调整回收计划或在当前章节回收

# 六、严重程度与处理流程总览

## 6.1 严重程度定义

| 级别 | 标记 | 定义 | 处理方式 | 示例 |
|------|------|------|----------|------|
| **硬伤** | 🔴 | 直接矛盾、逻辑不通、违反已建立规则 | 必须修复后才能继续 | 角色死而复生、修为倒退 |
| **中等** | 🟡 | 不够合理、可能引起读者困惑 | 建议修复，可后续修正 | 称呼不一致、时间间隔不合理 |
| **轻微** | 🟢 | 细节瑕疵、不影响大局 | 可选修复，记录在案 | 描写略有出入 |

## 6.2 处理流程

```
发现问题
    │
    ├─ 硬伤 → 立即修复 → 重新检查 → 确认通过 → 继续创作
    │
    ├─ 中等 → 记录问题 → 建议修复方案 → 用户决定修复时机
    │
    └─ 轻微 → 记录问题 → 供用户参考
```

# 七、与其他工具的协作关系

## 7.1 工具协作总览

| 工具 | 与 novel-manage 的协作方式 |
|------|---------------------------|
| novel-inspiration | 灵感阶段自动创建世界观初稿 |
| novel-outline | 大纲阶段扩展世界观（势力/地理/历史/等级） |
| novel-outline | 章节大纲阶段标注首次出现章节，添加伏笔追踪 |
| novel-chapter | 正文生成前读取世界观和写作反馈，生成后更新世界观/伏笔/进度 |
| novel-chapter | 每章/每批次完成后自动更新进度，调用一致性检查，生成写作反馈 |
| novel-quality | 世界观一致性影响评分 |
| novel-writing | 写作时引用世界观设定 |
| novel-deslop | 去AI味时保持世界观一致性 |

## 7.2 数据流向

```
灵感 ──→ 世界观初稿 ──→ 大纲扩展 ──→ 章节大纲标注 ──→ 正文生成
                ↑                                              │
                └──────── 每章完成后自动更新 ←──────────────────┘
                               │
                               ├─ 更新角色状态
                               ├─ 更新伏笔状态
                               ├─ 更新进度记录
                               └─ 触发一致性检查
                                       │
                                       ▼
                               生成写作反馈文件
                                       │
                                       ▼
                          ┌──→ 写作前读取反馈约束 ──→ 下一章正文生成
                          │            │
                          └────────────┘
                           （反馈循环：管理检查 → 反馈 → 写作 → 检查）
```

### 反馈循环说明

管理工具的一致性检查结果不再只是报告，而是转化为**可执行的写作约束**，通过 `追踪/写作反馈.md` 文件传递给写作流程。写作工具在生成正文前必须读取此文件，将反馈中的高频问题、角色注意事项、伏笔预警等作为写作约束，从而实现"管理→写作"的闭环反馈。

## 7.3 创作工具引用规则

**所有创作工具在生成内容时必须遵守以下规则**：

1. **必须读取世界观文档**：生成任何内容前，必须先读取 `设定/世界观.md`
2. **必须读取写作反馈**：生成正文前，必须读取 `追踪/写作反馈.md`，将反馈中的高频问题、角色注意事项、伏笔预警等作为写作约束
3. **不能违反已建立规则**：生成的内容不能与世界观文档中的设定矛盾
4. **必须更新世界观文档**：生成内容后，必须更新世界观文档中的相关条目
5. **必须记录新伏笔**：如果生成了伏笔，必须记录到伏笔追踪表
6. **必须更新进度**：生成正文后，必须更新进度记录
