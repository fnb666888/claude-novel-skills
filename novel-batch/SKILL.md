# 小说批量创作工具

**核心理念**：写一本150万字的小说不需要手动运行500次，批量工具帮你自动完成。

---

## 使用方式

### 批量生成章节大纲
- `/novel-batch 章节大纲 [整体大纲文件路径]` — 生成全书所有章节大纲
- `/novel-batch 章节大纲 [大纲文件路径] --from 第X章` — 从指定章节开始
- `/novel-batch 章节大纲 [大纲文件路径] --batch 50` — 每批生成50章（默认50）

### 批量生成正文
- `/novel-batch 正文 [整体大纲文件路径]` — 生成全书所有正文
- `/novel-batch 正文 [大纲文件路径] --from 第X章` — 从指定章节开始
- `/novel-batch 正文 [大纲文件路径] --batch 20` — 每批生成20章（默认20）

### 断点续写
- `/novel-batch 继续 [进度文件路径]` — 从上次中断处继续
- `/novel-batch 状态 [小说名]` — 查看当前进度

### 质量控制
- `/novel-batch 检查 [小说名]` — 检查已生成内容的一致性
- `/novel-batch 重新生成 [小说名] 第X章` — 重新生成指定章节

## 标准工作流

### 第一阶段：准备工作

```
1. 确保已有文件：
   - 灵感文件（inspiration_*.md）
   - 整体大纲文件（outline_*.md）
   - 世界观文件（world_*.md）

2. 检查就绪状态：
   - 大纲是否完整（八大部分）
   - 世界观是否已创建
   - 输出目录是否正确
```

### 第二阶段：批量生成章节大纲

```
1. 读取整体大纲和世界观设定
2. 按批次调用 /novel-chapter-outline 生成章节大纲（每批10章）
   - /novel-chapter-outline 内置评估+优化循环，batch 不重复处理
3. 每批完成后更新世界观文档
4. 保存进度文件
5. 输出进度报告
```

### 第三阶段：批量生成正文

```
1. 读取章节大纲和世界观设定
2. 按批次循环调用 /novel-chapter 生成正文（每批10章）
   - /novel-chapter 内置6阶段流水线（构思→调研→写作→评估→优化→去AI味），batch 不重复处理
3. 每章完成后：
   - 更新世界观文档（角色状态、事件记录）
   - 调用 /novel-consistency 检查连续性（每批次检查一次）
4. 保存进度文件
5. 输出进度报告
```

## 进度管理

### 进度文件格式

```markdown
# 批量创作进度：[小说名]

## 基本信息
- 小说名：[名称]
- 目标章节数：[X章]
- 目标字数：[X万字]
- 开始时间：[YYYY-MM-DD HH:MM]
- 最后更新：[YYYY-MM-DD HH:MM]

## 章节大纲进度
- 已生成：X/Y章
- 状态：进行中/已完成
- 最后生成：第X章

## 正文进度
- 已生成：X/Y章
- 已完成：X章（评分>=8.5）
- 需优化：X章（评分<8.5）
- 状态：进行中/已完成
- 最后生成：第X章

## 质量统计
- 平均评分：X.X/10
- 最高评分：X.X（第X章）
- 最低评分：X.X（第X章）
- 评分分布：
  - 9.0+：X章
  - 8.0-8.9：X章
  - 7.0-7.9：X章
  - <7.0：X章

## 世界观更新日志
| 章节 | 更新内容 |
|------|----------|
| 第X章 | [更新了什么] |
```

### 进度文件路径
- 默认：`~/小说/progress_[小说名].md`（跨平台兼容）
- 每次生成后自动更新

## 批量生成规则

### 章节大纲批量生成规则

| 规则 | 说明 |
|------|------|
| 批次大小 | 默认每批10章 |
| 承上启下 | 每批第1章必须接续上一批最后一章 |
| 评分阈值 | 平均分>=8.0才进入下一批 |
| 世界观更新 | 每批生成后更新世界观文档 |
| 进度保存 | 每批完成后保存进度 |

### 正文批量生成规则

| 规则 | 说明 |
|------|------|
| 批次大小 | 默认每批10章 |
| 连续性检查 | 每章生成后检查与前文的连续性 |
| 评分阈值 | 单章>=8.5才算完成，否则优化一轮 |
| 世界观更新 | 每章生成后更新世界观文档 |
| 字数控制 | 每章3500-4500字 |
| 进度保存 | 每章完成后保存进度 |

---

## 质量控制

### 自动质量检查

每章生成后自动检查：

| 检查项 | 标准 | 不合格处理 |
|--------|------|-----------|
| 评分 | >=8.5 | 自动优化一轮 |
| 字数 | 3500-4500 | 提示用户 |
| 连续性 | 无断裂 | 重新生成 |
| 世界观一致性 | 无矛盾 | 重新生成 |

### 手动干预

用户可以随时：
- 暂停批量生成
- 查看当前进度和质量统计
- 重新生成某章
- 调整后续章节的设计

## 断点续写机制

### 中断恢复

如果批量生成被中断：

```
1. 读取进度文件
2. 确定最后完成的章节
3. 读取该章节的内容和世界观状态
4. 从下一章继续生成
```

### 进度文件更新时机

- 每章完成后
- 每批次完成后
- 用户暂停时

## 与其他工具的协作

| 工具 | 协作方式 |
|------|----------|
| novel-chapter-outline | 批量调用生成章节大纲（内置评估+优化循环） |
| novel-chapter | 批量调用生成正文（内置6阶段流水线：构思→调研→写作→评估→优化→去AI味） |
| novel-consistency | 每批次完成后检查连续性 |
| novel-worldbuilding | 每章生成后更新世界观设定 |

> **说明**：评估（novel-evaluation）、优化（novel-optimization）、去AI味（novel-deslop）由 novel-chapter 内部调用，batch 不直接调用这些工具。
