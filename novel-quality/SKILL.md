# 小说质量闭环系统

融合评价与优化为一体：先评估"这章完成叙事任务了吗？"，再针对短板增量优化，迭代循环直到达标。

## 执行前置步骤

**重要：每次执行本skill时，必须先读取novel-writing技能的内容作为评价标准与优化工具箱。**

```
步骤1：读取 novel-writing/SKILL.md（按角色/情节/写作选择对应模块：references/module-a-character.md、module-b-plot.md、module-c-prose.md）
步骤2：读取去耦合分类库（按需读取）：
  - novel-writing/references/character-traits.md — 角色特点分类（个性×情绪×行为×语言）
  - novel-writing/references/plot-elements.md — 情节元素分类（事件类型×事件风格×场景氛围）
  - novel-writing/extensions/ — 扩展分类库（如有）
步骤3：判定输入类型（灵感/大纲/正文），确定评价维度和权重
步骤4：执行评价 → 优化 → 再评价的迭代闭环
```

- `/novel-quality 评分 大纲 [内容或文件路径]` — 评价大纲的节奏设计

- `/novel-quality 评分 灵感/大纲/正文 --readers` — 输出读者反馈模拟报告

- `/novel-quality 优化 灵感 [内容或文件路径]` — 优化灵感
- `/novel-quality 优化 大纲 [内容或文件路径]` — 优化大纲
- `/novel-quality 优化 正文 [内容或文件路径]` — 优化正文

- `/novel-quality 优化 灵感/大纲/正文 --eval [评价报告文件路径]` — 根据评价报告定向优化

## 评价哲学

### 核心转变

**旧框架**：检查"有没有某个元素"（亲密场景？✓/✗，笑点？数量）
**新框架**：评估"目标是否达成"（这章让读者想看下一章了吗？）

### 极端化原则（核心）

**好小说不是各维度都高分，而是某个维度做到极致。**

- 暧昧情节就要足够暧昧——让人心跳加速到脸红
- 性爱情节就要足够诱惑——让人血脉贲张
- 悲伤情节就要足够悲伤——让人泪流不止
- 搞笑情节就要足够搞笑——让人笑出声来
- 悬疑情节就要足够悬疑——让人猜不到结局
- 热血情节就要足够热血——让人热血沸腾

**评价方式**：
1. 先声明本内容的**主要体验类型**（暧昧/诱惑/悲伤/搞笑/悬疑/热血/治愈/...）
2. 核心体验维度做到 **9+** 即为优秀，其他维度 **不拖后腿**（≥5）即可
3. **允许偏科**——与其各维度都7分，不如一个维度9.5+、其他维度6

**极端化评估**（贯穿所有量表）：

| 内容类型 | 核心极端维度 | 极端化的标准 |
|----------|------------|------------|
| 暧昧情节 | 暧昧感 | 让读者脸红心跳，反复回味暧昧细节 |
| 诱惑情节 | 诱惑力 | 让读者血脉贲张，欲罢不能 |
| 悲伤情节 | 悲伤感 | 让读者泪流不止，久久不能释怀 |
| 搞笑情节 | 笑点密度 | 让读者笑出声来，停不下来 |
| 悬疑情节 | 悬念感 | 让读者猜不到结局，欲罢不能 |
| 热血情节 | 热血感 | 让读者热血沸腾，握紧拳头 |
| 甜蜜情节 | 甜蜜度 | 让读者甜到牙疼，嘴角上扬 |
| 治愈情节 | 治愈感 | 让读者感到温暖，内心平静 |
| 高潮情节 | 冲击力 | 让读者"卧槽"，目瞪口呆 |
| 日常情节 | 烟火气 | 让读者有代入感，会心一笑 |

### 去耦合化评估原则（核心）

**好小说的角色和情节不是绑定的，而是自由组合的。**

评价时检查：
1. **角色维度多样性**：同一角色在不同场景中是否展现了不同的情绪×行为×语言组合？（而非永远"高冷"）
2. **情节元素独立性**：事件类型、事件风格、场景氛围是否独立选择？（而非"误会"永远配"虐心"）
3. **交叉组合新鲜感**：角色特点与情节元素的组合是否给读者带来意外？（而非按套路搭配）

**优化时的去耦合化策略**：
- 角色感觉扁平 → 不是"加背景故事"，而是**更换情绪×行为×语言组合**（参考 character-traits.md）
- 情节感觉套路 → 不是"加反转"，而是**更换事件类型×风格×氛围组合**（参考 plot-elements.md）
- 优化示例：不是说"用反差感塑造法"，而是说"试试 傲娇型×委屈情绪×咬唇行为×毒舌语言"

### 优化哲学

1. **极端化优先** — 先确定内容的自然优势，将其推到极致，而非面面俱到
2. **偏科策略** — 允许某维度9.5+而其他维度6，优于各维度都7.5
3. **增量优化** — 保留原有内容的核心魅力，只增强薄弱环节
4. **技法驱动** — 所有优化建议必须引用novel-writing中的具体技法
5. **去耦合驱动** — 优化建议优先通过更换去耦合元素组合来突破瓶颈，引用 character-traits.md / plot-elements.md 中的具体元素
6. **可执行** — 输出优化后的完整内容，不是抽象建议
7. **吸引力导向** — 优化目标是"读者想看"，不是"写得好"

### 三层层级评价

| 层级 | 权重 | 核心问题 | 说明 |
|------|------|----------|------|
| 第一层：章节目标达成度 | 40% | 这章完成它的叙事任务了吗？ | 根据章节类型动态调整评价标准 |
| 第二层：读者体验维度 | 40% | 读者读这章时的体验如何？ | 沉浸感、情绪波动、翻页欲 |
| 第三层：技巧执行度 | 20% | 写作技巧是否到位？ | 描写质量、对话质量、节奏控制、金句记忆 |

**极端化修正**：如果核心体验维度得分 ≥ 9.5，即使其他维度略低（≥5），总分也可上浮 0.5-1.0 分。

### 评价尺度

| 分数 | 含义 | 读者反应 |
|------|------|----------|
| 9-10 | 致命吸引力 | "卧槽太好看了，通宵看完" |
| 7-8 | 强吸引力 | "挺有意思的，想继续看" |
| 5-6 | 一般吸引力 | "还行吧，有空再看" |
| 3-4 | 弱吸引力 | "看不太进去，有点无聊" |
| 1-2 | 无吸引力 | "弃了" |

### 优化优先级

| 优先级 | 维度 | 原因 |
|--------|------|------|
| P0 | 核心体验极端化 | 让内容在某个维度做到极致 |
| P1 | 亲密关系 | 直接决定读者是否继续看 |
| P2 | 幽默趣味 | 娱乐性是网文核心竞争力 |
| P3 | 惊喜意外 | 防止审美疲劳 |
| P4 | 烟火日常 | 增强代入感和真实感 |
| P5 | 翻页欲 | 悬念驱动追更 |
| P6 | 角色魅力 | 长期留存的关键 |

## 迭代闭环流程

```
声明核心体验 → 生成 → 评分（极端化评估）→ 优化（极端化优先）→ 再评分 → 直到核心体验≥9.5 或总分≥9.0
```

### 执行步骤

1. **声明核心体验**：确定本内容的主要体验类型（暧昧/悲伤/搞笑/热血/悬疑/甜蜜/...）
2. **评分**：对灵感/大纲/正文执行评价，输出评价报告，标注核心体验维度得分
3. **极端化评估**：核心体验维度是否 ≥ 9.5？是则直接通过
4. **优化**：
   - **核心体验不够极端** → 集中火力将其推到极致
   - **核心体验已极端但其他维度拖后腿**（<5分） → 修补明显短板
   - **两者都不满足** → 先极端化核心体验，再修补短板
5. **去AI味**：正文优化后自动运行novel-deslop（Gate A-F扫描 + 三遍法修复）
6. **再评分**：对优化后的内容重新评价
7. **判定**：核心体验 ≥ 9.5 或总分 ≥ 9.0 则输出最终版本；否则回到步骤4继续迭代
8. **上限**：最多迭代3轮，避免过度优化导致内容失真

## 量表A：灵感评价

评估灵感概念的吸引力潜力，共6个维度。

### 维度定义

| 维度 | 权重 | 核心问题 | 参照novel-writing |
|------|------|----------|-------------------|
| 亲密潜力 | 30% | 这个设定能产生多少让人心跳加速的亲密场景？ | 模块C：亲密场景写作 |
| 反差魅力 | 25% | 角色设定是否有"没想到"的惊喜感？ | 模块A：反差感塑造法 |
| 趣味基因 | 20% | 这个设定本身是否有趣/好笑？ | 模块C：幽默技巧 |
| 意外系数 | 15% | 有多少出人意料的可能性？ | 模块A：圣体系统、模块B：信息差 |
| 日常可塑性 | 5% | 能否写出有烟火气的日常场景？ | 模块C：场景构建 |
| 上瘾钩子 | 5% | 核心设定是否自带"想知道后续"的驱动力？ | 模块B：钩子设置技巧 |

### 逐维度评价指引

**亲密潜力（30%）**
- 9-10分：设定天然包含禁忌感/身份差/被迫亲密等要素，能自然产生大量高张力亲密场景（参考：师姐师弟、仙子凡人、女帝臣子）
- 7-8分：设定有亲密空间，需要一些情节设计来触发
- 5-6分：亲密场景需要刻意制造，不够自然
- 3-4分：设定与亲密关系关联弱
- 1-2分：几乎没有亲密场景的可能性

**反差魅力（25%）**
- 9-10分：核心角色有强烈的反差设计（外在vs内在），能产生"清冷女神私下居然是..."的效果
- 7-8分：有反差设计，效果明显
- 5-6分：有反差但不够强烈
- 3-4分：反差弱或过于套路
- 1-2分：角色扁平无反差

**趣味基因（20%）**
- 9-10分：设定本身自带喜感（如"合欢宗的老韭菜"），能自然产生喜剧效果
- 7-8分：有趣味空间，有喜剧潜力
- 5-6分：趣味性一般，需要刻意制造
- 3-4分：设定严肃，趣味性弱
- 1-2分：完全没有趣味基因

**意外系数（15%）**
- 9-10分：设定包含多个出人意料的元素（身份反转、能力隐藏、关系错位），能持续制造惊喜
- 7-8分：有意外元素，能制造几次大反转
- 5-6分：意外元素少，主要靠情节推进
- 3-4分：设定中规中矩，缺乏惊喜
- 1-2分：完全套路化

**日常可塑性（5%）**
- 9-10分：设定能自然产生大量有烟火气的日常场景（吃饭、逛街、居家）
- 7-8分：有日常场景空间
- 5-6分：日常场景需要刻意设计
- 3-4分：设定与日常生活脱节
- 1-2分：无法产生日常场景

**上瘾钩子（5%）**
- 9-10分：核心设定自带"想知道接下来会怎样"的驱动力（如死亡倒计时、身份秘密）
- 7-8分：有悬念驱动力
- 5-6分：悬念一般
- 3-4分：缺乏悬念
- 1-2分：没有钩子

### 极端化评估（量表A核心）

**在评分前，先声明本灵感的核心体验类型**：

| 灵感类型 | 核心维度 | 极端化标准 |
|----------|---------|-----------|
| 暧昧型 | 亲密潜力 | 让读者看到设定就脸红心跳 |
| 反差型 | 反差魅力 | 让读者看到角色就"没想到" |
| 搞笑型 | 趣味基因 | 让读者看到设定就笑出来 |
| 悬疑型 | 意外系数 | 让读者看到设定就想知道真相 |
| 烟火型 | 日常可塑性 | 让读者看到设定就想到生活场景 |
| 上瘾型 | 上瘾钩子 | 让读者看到设定就停不下来 |

**极端化评分修正**：
- 核心维度 ≥ 9.5 → 总分上浮 0.5-1.0
- 核心维度 ≥ 9.0 且其他维度无 < 5 → 总分上浮 0.3
- 核心维度 < 7.0 → 即使总分高，也需优化核心维度

**示例**：
- "合欢宗的处男炼丹师"→ 核心类型：暧昧型 + 搞笑型
  - 亲密潜力 9.5 + 趣味基因 9.0 → 核心维度达标，即使其他维度一般也可通过
- "修仙世界的普通弟子"→ 核心类型不明确
  - 各维度都在 6-7 分 → 没有极端化优势，需要找到核心卖点并推到极致

### 灵感优化技法

| 维度 | 优化技法（来自novel-writing） | 去耦合化优化策略 | 优化目标 |
|------|-------------------------------|-----------------|----------|
| 亲密潜力 | 模块A：反差感塑造法、模块C：身份禁忌极致化 | 更换角色的情绪×行为组合（如：高冷型×害羞情绪×咬唇行为 → 制造亲密张力） | 让设定天然包含高张力亲密元素 |
| 反差魅力 | 模块A：核心六要素、模块A：表里双层叙事法 | 同一角色在公开/私人场景使用不同个性×语言组合（参考 character-traits.md 对比组合技法） | 让角色有"没想到"的惊喜感 |
| 趣味基因 | 模块A：一句话定义法、模块C：幽默技巧 | 更换事件风格×场景氛围（如：日常对话×搞笑幽默×尴尬喜剧 → 天然喜感） | 让设定本身自带喜感 |
| 意外系数 | 模块A：圣体系统、模块B：信息差驱动法 | 更换事件类型×事件风格组合（如：身份揭露×反转风格×紧张氛围 → 惊喜） | 增加出人意料的可能性 |
| 日常可塑性 | 模块C：场景构建、模块C：感官叠加 | 选择烟火气事件类型×治愈风格×日常氛围（参考 plot-elements.md 模式配方） | 让设定能产生烟火气场景 |
| 上瘾钩子 | 模块B：钩子设置技巧、模块B：嵌套式叙事框架 | 每章更换事件类型×风格组合，保持新鲜感 | 让设定自带驱动力 |

### 灵感优化示例

**原始灵感**：
> 一个修仙世界的炼丹师，天赋平平但努力修炼。

**短板分析**：
- 亲密潜力弱：没有禁忌感/身份差
- 反差魅力弱：角色设定扁平
- 趣味基因弱：没有喜感元素

**优化版灵感**：
> 合欢宗的炼丹师，专门炼制双修丹药，但自己却是个处男。每次给师姐们送丹药都要被调戏，偏偏他的丹药效果最好，师姐们都指名要他。
>
> 反差点：外表老司机，内心纯情小处男
> 亲密潜力：送丹药=被迫亲密，师姐调戏=自然亲密场景
> 趣味基因："合欢宗的处男炼丹师"自带喜感
> 圣体属性：先天纯阳圣体（炼丹天赋极高，但对媚术免疫）

## 量表B：大纲评价

评估大纲设计的吸引力节奏，共7个维度。

### 维度定义

| 维度 | 权重 | 核心问题 | 参照novel-writing |
|------|------|----------|-------------------|
| 亲密节奏 | 30% | 亲密场景的分布是否合理？是否有渐进式升级？ | 模块C：渐进式隐喻系统、模块B：感情线构造 |
| 日常密度 | 15% | 有多少让人会心一笑的日常场景？ | 模块C：场景构建 |
| 幽默分布 | 15% | 笑点是否均匀分布？是否有"减压阀"？ | 模块C：幽默技巧、模块C：叙事节奏控制 |
| 惊喜节点 | 15% | 在哪些位置设置了出人意料的转折？ | 模块B：钩子设置技巧、模块A：表里双层叙事法 |
| 节奏控制 | 10% | 快慢交替是否合理？是否有张弛？ | 模块C：叙事节奏控制 |
| 悬念密度 | 10% | 有多少"想知道后续"的钩子？ | 模块B：钩子设置技巧 |
| 情感峰值 | 5% | 是否有让人泪目/心动的高潮点？ | 模块B：高潮场景设计 |

### 逐维度评价指引

**亲密节奏（30%）**
- 9-10分：亲密场景遵循"渐进式升级"（小雨→中雨→大雨→雷鸣），分布均匀，每次亲密都比上次更进一步，有明确的"克制→突破"节奏
- 7-8分：亲密场景有阶段划分，基本遵循渐进逻辑
- 5-6分：亲密场景存在但节奏不够清晰
- 3-4分：亲密场景生硬或突兀
- 1-2分：几乎没有亲密场景设计

**日常密度（15%）**
- 9-10分：每3-5章就有一次有烟火气的日常场景（做饭、逛街、居家互动），日常中自然推进感情
- 7-8分：日常场景分布合理
- 5-6分：日常场景偏少
- 3-4分：几乎没有日常场景
- 1-2分：完全没有生活感

**幽默分布（15%）**
- 9-10分：笑点分布均匀，每2-3章至少一个笑点，有"减压阀"功能（紧张后必有轻松），幽默类型多样
- 7-8分：幽默分布合理，有笑点
- 5-6分：幽默偏少或分布不均
- 3-4分：幽默生硬
- 1-2分：几乎没有幽默

**惊喜节点（15%）**
- 9-10分：在20%/50%/80%位置设置重大反转，有身份揭露/关系逆转/意外发展，每个转折都让人"卧槽"
- 7-8分：有2-3个惊喜节点
- 5-6分：惊喜节点少
- 3-4分：缺乏惊喜
- 1-2分：完全套路化

**节奏控制（10%）**
- 9-10分：快慢交替自然，高压段（亲密/冲突）后必有低压段（日常/幽默），节奏感强
- 7-8分：节奏基本合理
- 5-6分：节奏有但不够均匀
- 3-4分：节奏单调
- 1-2分：没有节奏控制

**悬念密度（10%）**
- 9-10分：每个章节结尾都有钩子，类型多样（悬念型/转折型/信息揭露型），读者停不下来
- 7-8分：大部分章节有钩子
- 5-6分：钩子偏少
- 3-4分：钩子稀疏
- 1-2分：几乎没有钩子

**情感峰值（5%）**
- 9-10分：有2-3个让人泪目/心动的高潮点，分布合理，情感冲击力强
- 7-8分：有情感峰值
- 5-6分：情感峰值弱
- 3-4分：缺乏情感高潮
- 1-2分：没有情感设计

### 极端化评估（量表B核心）

**在评分前，先声明本大纲的核心体验类型**：

| 大纲类型 | 核心维度 | 极端化标准 |
|----------|---------|-----------|
| 甜宠型 | 亲密节奏 | 每3-5章一次心动场景，渐进升级让人停不下来 |
| 搞笑型 | 幽默分布 | 每2-3章一个笑点，笑到停不下来 |
| 悬疑型 | 惊喜节点 + 悬念密度 | 反转不断，猜不到下一步 |
| 热血型 | 情感峰值 + 节奏控制 | 高潮迭起，热血沸腾 |
| 治愈型 | 日常密度 + 情感峰值 | 温暖日常，泪中带笑 |

**极端化评分修正**：
- 核心维度 ≥ 9.5 → 总分上浮 0.5-1.0
- 核心维度 ≥ 9.0 且其他维度无 < 5 → 总分上浮 0.3
- 核心维度 < 7.0 → 即使总分高，也需优化核心维度

**示例**：
- 甜宠大纲：亲密节奏 9.5，幽默 7.0，悬念 6.5 → 核心维度达标，可通过
- 悬疑大纲：惊喜节点 6.0，悬念 7.0 → 核心维度不达标，需优化反转设计

### 大纲优化技法

| 维度 | 优化技法（来自novel-writing） | 去耦合化优化策略 | 优化目标 |
|------|-------------------------------|-----------------|----------|
| 亲密节奏 | 模块C：渐进式隐喻系统、模块B：渐进式推进法 | 每次亲密场景更换事件风格×氛围组合（如：小雨=暧昧拉扯×私密氛围 → 大雨=禁忌刺激×浪漫仪式） | 亲密场景分布合理，有升级感 |
| 日常密度 | 模块C：场景构建、模块C：感官叠加 | 日常场景使用不同事件类型×风格组合（参考 plot-elements.md 事件类型库：日常陪伴/日常对话/日常喜剧轮换） | 增加烟火气场景 |
| 幽默分布 | 模块C：幽默技巧、模块C：破折号旁白法 | 不同角色使用不同语言风格×行为组合制造幽默（参考 character-traits.md 语言风格库） | 笑点均匀分布 |
| 惊喜节点 | 模块B：钩子设置技巧、模块A：表里双层叙事法 | 惊喜节点使用非常规事件类型×风格组合（如：身份揭露×反转风格 配 治愈氛围 → 反差惊喜） | 设置出人意料的转折 |
| 节奏控制 | 模块C：叙事节奏控制 | 通过事件风格交替控制节奏（甜→虐→甜，参考 plot-elements.md 8种情节模式配方） | 快慢交替合理 |
| 悬念密度 | 模块B：钩子设置技巧、模块B：信息差驱动法 | 每章结尾更换悬念类型（信息差/关系变化/身份暗示），不重复同一套路 | 每章都有钩子 |
| 情感峰值 | 模块B：高潮场景设计、模块C：以死换情法 | 高潮场景使用角色的非常规情绪×行为组合（如：平时毒舌的角色此刻沉默→冲击力更强） | 设置让人泪目的高潮 |

### 大纲优化示例

**原始大纲片段**：
> 第5章：主角和女主角一起吃饭，聊了聊最近的情况。
> 第6章：主角去修炼，实力提升。
> 第7章：遇到敌人，发生战斗。

**短板分析**：
- 第5章：日常场景但缺乏亲密和幽默
- 第6章：纯修炼，节奏单调
- 第7章：战斗后缺乏减压

**优化版大纲**：
> 第5章：主角和女主角一起吃饭，女主角不小心把汤洒在衣服上，主角帮她擦，两人距离突然拉近（亲密升温：小雨级别）。女主角嘴硬说"我自己来"，但脸已经红了（反差萌+幽默）。
>

>
> 第7章：遇到敌人，发生战斗。战斗中女主角受伤，主角抱着她逃跑（亲密+紧张）。章末钩子："她的伤只有双修才能治"（悬念+亲密预告）。

### 正文评价策略

**第一步：判定章节类型 + 声明核心体验**
1. 阅读章节内容，判定属于哪种类型（开篇章/亲密章/日常章/高潮章/转折章/暧昧章/悲伤章/搞笑章/诱惑章/治愈章）
2. 声明本章的核心极端维度（如暧昧章→暧昧感极致度）
3. 确定该类型的评价维度和权重

**第二步：定位关键片段**
1. 根据章节类型，定位对应的关键片段：
   - 开篇章：世界观介绍、角色出场、章末钩子
   - 亲密章：亲密场景、情感爆发点
   - 日常章：生活细节、角色互动
   - 高潮章：冲突场景、反转时刻
   - 转折章：信息揭露、剧情转向
   - 暧昧章：暧昧互动、若即若离的细节
   - 悲伤章：离别场景、情感爆发点
   - 搞笑章：笑点、吐槽、反差场景
   - 诱惑章：亲密诱惑、欲拒还迎的细节
   - 治愈章：温暖场景、和解时刻

**第三步：逐层评价 + 极端化评估**
1. **第一层**：根据章节类型，评价目标达成度
2. **第二层**：评价读者体验（沉浸感、情绪波动、翻页欲）
3. **第三层**：评价技巧执行度（描写、对话、节奏、金句）
4. **极端化评估**：核心极端维度是否 ≥ 9.5？是否做到了极致？

**第四步：计算总分（含极端化修正）**
```
基础总分 = 第一层得分 × 40% + 第二层得分 × 40% + 第三层得分 × 20%

极端化修正：
- 核心极端维度 ≥ 9.5 → 总分上浮 0.5-1.0（最高10.0）
- 核心极端维度 ≥ 9.0 且其他维度无 < 5 → 总分上浮 0.3
- 核心极端维度 < 7.0 → 标记"核心体验不足"，即使总分高也需优化
```

### 正文优化技法

**极端化优先策略**：先确定核心体验，集中火力将其推到极致，再修补明显短板。

| 核心体验 | 极端化技法（来自novel-writing） | 极端化目标 |
|----------|-------------------------------|-----------|
| 暧昧感极致化 | 模块C：感官分层法 + 克制留白法 + 渐进式隐喻法 | 让读者脸红心跳，反复回味 |
| 诱惑力极致化 | 模块C：感官分层法 + 环境烘托法 + 微反应刻画法 | 让读者血脉贲张，欲罢不能 |
| 悲伤感极致化 | 模块C：重复递进情绪法 + 以死换情法 + 环境烘托法 | 让读者泪流不止，久久不能释怀 |
| 笑点极致化 | 模块C：内心独白式幽默 + 对比式幽默 + 破折号旁白法 | 让读者笑出声来，停不下来 |
| 悬念极致化 | 模块B：信息差驱动法 + 钩子设置技巧 + 嵌套式叙事框架 | 让读者猜不到结局，欲罢不能 |
| 热血极致化 | 模块B：高潮场景设计 + 模块C：重复递进情绪法 | 让读者热血沸腾，握紧拳头 |
| 甜蜜极致化 | 模块C：感官叠加法 + 微反应刻画法 + 环境烘托法 | 让读者甜到牙疼，嘴角上扬 |
| 治愈极致化 | 模块C：场景构建 + 感官叠加法 + 生活细节法 | 让读者感到温暖，内心平静 |

**通用优化技法**（修补短板用）：

| 维度 | 优化技法（来自novel-writing） | 去耦合化优化策略 | 优化目标 |
|------|-------------------------------|-----------------|----------|
| 亲密描写 | 模块C：亲密场景写作全部技法 | 更换角色的情绪×行为×语言组合（如：从"害羞咬唇"换成"大胆撩拨"→制造反差） | 让亲密描写让人心跳加速 |
| 烟火气 | 模块C：场景构建、模块C：感官叠加 | 选择烟火气事件类型×治愈/日常风格×日常氛围（参考 plot-elements.md 情节模式配方） | 让日常场景有生活感 |
| 笑点密度 | 模块C：幽默技巧全部技法 | 不同角色用不同语言风格×行为组合制造幽默（参考 character-traits.md 语言风格库） | 让人笑出来 |
| 惊喜感 | 模块A：表里双层叙事法、模块B：信息差 | 更换事件类型×风格组合（如：日常对话场景突然切反转风格→意外） | 让人"卧槽" |
| 翻页欲 | 模块B：钩子设置技巧 | 每章结尾更换悬念类型，不重复套路 | 让人停不下来 |
| 角色魅力 | 模块A：核心六要素全部 | 为角色选择独特的个性×情绪×行为×语言组合（参考 character-traits.md 组合技法） | 让人上头 |
| 代入感 | 通用质量检查 | 使用读者熟悉的情绪×行为组合（参考 character-traits.md 情绪类型库的"典型触发条件"） | 让人代入 |
| 金句记忆 | 模块C：金句重复法 | 金句与角色的语言风格类型匹配（参考 character-traits.md 语言风格库） | 让人想截图 |
| 去AI味 | novel-deslop：6 Gate + 三遍法 | 让文字像人写的 |

## 正文优化示例

### 亲密描写优化

**原始**：
> 他吻了她。

**优化**：
> 他的唇轻轻贴上她的，温热的触感让她浑身一颤。先是蜻蜓点水般的试探，感受到她没有抗拒，才逐渐加深这个吻。她的睫毛微微颤动，呼吸变得急促，手指不自觉地抓紧了他的衣角。

### 幽默优化

**原始**：
> 他很尴尬。

**优化**：
> 他站在那里，感觉自己像个被老师点名回答问题却什么都不会的小学生。不对，小学生至少还能装肚子疼，他现在连装的资格都没有。

### 惊喜优化

**原始**：
> 她其实是大小姐。

**优化**：
> 她穿着围裙，在小餐馆里端盘子，被客人呼来喝去。他心疼地想帮她，却被她拒绝。
>
> 直到那天，一辆劳斯莱斯停在餐馆门口，管家恭敬地鞠躬："小姐，老爷请您回家。"
>
> 她头也不回："告诉老头子，我在体验生活。"
>
> 他站在原地，手里的筷子掉在地上。

### 日常细节优化

**原始**：
> 他们一起吃饭。

**优化**：
> 火锅的热气氤氲，模糊了她的脸。他夹起一片毛肚，在红油里七上八下，递到她碗里。她低头吹了吹，辣椒油溅到嘴角，她用手背擦了擦，又舔了舔嘴唇。
>
> "辣吗？"他问。
>
> 她点头，眼泪都出来了，却还在往嘴里塞。

## 执行规则

### 1. 总分计算（含极端化修正）
```
基础总分 = 第一层得分 × 40% + 第二层得分 × 40% + 第三层得分 × 20%
```
其中：
- 第一层得分 = Σ (维度得分 × 维度权重)
- 第二层得分 = 沉浸感 × 37.5% + 情绪波动 × 37.5% + 翻页欲 × 25%
- 第三层得分 = 描写质量 × 25% + 对话质量 × 25% + 节奏控制 × 20% + 去耦合多样性 × 15% + 金句记忆 × 15%

**极端化修正**：
- 声明核心体验类型，确定核心极端维度
- 核心极端维度 ≥ 9.5 → 总分上浮 0.5-1.0（最高10.0）
- 核心极端维度 ≥ 9.0 且其他维度无 < 5 → 总分上浮 0.3
- 核心极端维度 < 7.0 → 标记"核心体验不足"

### 2. 证据要求
每个维度的评价必须附带至少一个具体引用：
- 灵感：引用具体设定
- 大纲：引用具体情节设计
- 正文：引用原文段落（标注位置）

### 3. 读者反应预测
每个维度都要给出"读者反应预测"——不是抽象评价，而是具体的读者行为预测：
- "读者会心跳加速"
- "读者会笑出声"
- "读者会截图分享"
- "读者会熬夜追更"

**核心体验反应预测**（必须包含）：
- 核心极端维度的读者反应必须具体到行为："读者会反复翻看这段"、"读者会截图发朋友圈"、"读者会看哭"

### 4. 等级判定
- 总分 >= 8.0 → **[优秀，值得开发]**
- 总分 >= 6.0 → **[良好，建议优化]**
- 总分 >= 4.0 → **[一般，需要优化]**
- 总分 < 4.0 → **[较差，需要大幅优化]**
- 核心极端维度 >= 9.5 → **[极致体验，即使总分不高也值得保留]**

### 5. 优化执行规则
- **极端化优先**：先确定内容的自然优势，将其推到极致，而非面面俱到
- **偏科策略**：允许某维度9.5+而其他维度6，优于各维度都7.5
- **保留核心魅力**：优化不是重写，是增强。必须保留原始内容的核心卖点和独特魅力
- **去耦合化重组**：优化时优先通过更换去耦合元素组合来突破瓶颈（参考 character-traits.md / plot-elements.md），而非简单地"加料"
- **技法引用要求**：每个优化点必须引用novel-writing中的具体技法（技法名称、所在模块A/B/C、核心要点）
- **去耦合引用要求**：涉及角色表现或情节设计的优化点，必须引用 character-traits.md 或 plot-elements.md 中的具体元素组合
- **修改前后对比**：每个优化点必须展示修改前后的对比
- **优化幅度控制**：灵感可大幅调整但保留核心设定；大纲以结构调整为主保留关键情节；正文段落级修改保留整体框架
- **短板修补**：只修补 < 5 分的明显短板，不追求所有维度都高分

### 6. 优化指令格式
优化指令必须：
- 引用novel-writing中的**具体技法名称**
- 说明该技法在novel-writing中的**大致位置**
- 引用 character-traits.md 或 plot-elements.md 中的**具体去耦合元素组合**（如"傲娇型×委屈情绪×咬唇行为×毒舌语言"）
- 针对当前内容给出**具体的修改方向**
- 预测优化后的**读者反应提升**

## 输出文档规则

**重要：每次运行本skill后，必须将评价/优化报告保存为md文档。**

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：
  - 灵感评价：`eval_inspiration_[时间戳].md`
  - 大纲评价：`eval_outline_[时间戳].md`
  - 正文评价：`eval_text_[时间戳].md`
  - 灵感优化：`opt_inspiration_[时间戳].md`
  - 大纲优化：`opt_outline_[时间戳].md`
  - 正文优化：`opt_text_[时间戳].md`
- 如用户指定目录则使用用户指定的目录

### 输出内容
- 评价报告：完整的评价报告内容，格式参照上方"输出格式"章节
- 优化报告：原始内容、吸引力短板分析、优化策略、优化版内容、优化点说明、预期读者反应提升

### 输出确认
保存文件后，向用户报告：
- 文件保存路径
- 评价模式：总分及等级判定、最大的1-2个吸引力优势、最需要优化的1-2个短板
- 优化模式：优化了哪些维度、预期的读者反应提升、优化版内容的核心亮点

*极端化原则：核心体验做到极致，允许偏科，不追求各维度均衡*
*去耦合化原则：角色特点×情节元素自由组合，优化时通过更换元素组合突破瓶颈*
*增量优化原则：保留核心魅力，只增强薄弱环节*
