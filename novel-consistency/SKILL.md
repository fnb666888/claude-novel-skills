# 小说连续性检查工具

**核心理念**：读者最怕的不是写得不好，而是前后矛盾。连续性检查帮你避免"硬伤"。

---

## 使用方式

### 全书检查
- `/novel-consistency 检查 [小说名]` — 检查全书的连续性
- `/novel-consistency 检查 [小说名] --from 第X章 --to 第Y章` — 检查指定范围

### 单章检查
- `/novel-consistency 检查章 [章节文件路径]` — 检查单章与前文的连续性
- `/novel-consistency 检查章 [章节文件路径] --previous [上一章文件路径]` — 指定上一章

### 专项检查
- `/novel-consistency 角色 [小说名]` — 只检查角色一致性
- `/novel-consistency 伏笔 [小说名]` — 只检查伏笔回收
- `/novel-consistency 时间线 [小说名]` — 只检查时间线
- `/novel-consistency 世界观 [小说名]` — 只检查世界观一致性

## 检查维度

### 1. 角色一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 性格一致性 | 角色行为是否符合已建立的性格 | 高 |
| 语言风格一致性 | 对话风格是否前后一致 | 中 |
| 能力一致性 | 角色能力是否超出设定 | 高 |
| 外貌一致性 | 外貌描述是否前后矛盾 | 中 |
| 称呼一致性 | 角色间称呼是否正确 | 中 |
| 状态一致性 | 角色当前状态（位置/伤势/情绪）是否正确 | 高 |

#### 检查方法

```
1. 建立角色档案（从世界观文档读取）
2. 扫描全书，提取每个角色的行为/对话/状态
3. 对比角色档案，发现不一致之处
4. 输出检查报告
```

#### 检查示例

```
问题：第3章描写林风眠"修为刚入炼气期"，但第15章他"轻松击败筑基期敌人"
→ 硬伤：修为设定矛盾
→ 建议：修改第15章，增加合理的解释（如使用金手指/法宝）
```

### 2. 伏笔回收

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 未回收伏笔 | 已埋设但未回收的伏笔 | 高 |
| 回收矛盾 | 回收方式与埋设时的暗示不符 | 高 |
| 重复伏笔 | 同一个伏笔被埋设多次 | 中 |

#### 检查方法

```
1. 从世界观文档读取伏笔追踪表
2. 扫描全书，标记伏笔的埋设和回收
3. 检查：
   - 埋设的伏笔是否都有回收
   - 回收方式是否与暗示一致
   - 是否有重复埋设
4. 输出检查报告
```

#### 检查示例

```
问题：第5章提到"双鱼佩在关键时刻会发光"，但全书再未提及
→ 未回收伏笔
→ 建议：在后续章节中回收（如第50章用双鱼佩打断亲密场景）
```

### 3. 时间线一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 时间顺序矛盾 | 事件发生的先后顺序不合理 | 高 |
| 时间跨度矛盾 | 两事件之间的时间间隔不合理 | 中 |
| 年龄矛盾 | 角色年龄与时间流逝不符 | 高 |
| 季节矛盾 | 季节/天气描述与时间不符 | 中 |

#### 检查方法

```
1. 建立时间线（从世界观文档和正文提取）
2. 标记每个事件的发生时间
3. 检查时间顺序和间隔
4. 输出检查报告
```

#### 检查示例

```
问题：第10章是"春天"，第12章是"三天后"却描写"漫天飞雪"
→ 季节矛盾
→ 建议：修改天气描述或时间间隔
```

### 4. 关系状态一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 关系进展 | 关系发展是否符合前文铺垫 | 高 |
| 称呼变化 | 称呼是否随关系变化而变化 | 中 |
| 情感状态 | 角色情感状态是否正确 | 高 |
| 多角关系 | 多条感情线是否冲突 | 中 |

#### 检查方法

```
1. 从世界观文档读取角色关系表
2. 扫描全书，标记关系变化节点
3. 检查关系进展是否合理
4. 输出检查报告
```

### 5. 世界观一致性

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 等级体系 | 等级描述是否前后一致 | 高 |
| 势力关系 | 势力间关系是否前后一致 | 高 |
| 地理距离 | 地点间距离/方位是否矛盾 | 高 |
| 规则违反 | 是否违反已建立的世界规则 | 高 |
| 物品追踪 | 关键物品的去向是否明确 | 中 |

#### 检查方法

```
1. 从世界观文档读取所有设定
2. 扫描全书，提取世界观相关名词
3. 逐项对比，发现矛盾
4. 输出检查报告
```

## 检查触发时机

### 自动触发
- 每章正文生成后（通过novel-batch）
- 批量生成每批次结束后

### 手动触发
- 用户运行 `/novel-consistency 检查`
- 用户编辑世界观设定后
- 用户修改已生成章节后

---

## 修复建议

### 硬伤修复
- 必须修复后才能继续生成
- 提供具体的修改建议
- 如果矛盾严重，建议重新生成该章节

### 中等问题修复
- 建议修复，但不强制
- 提供修改方向
- 可以在后续章节中自然修正

### 小问题修复
- 可选修复
- 记录在案，供用户参考
