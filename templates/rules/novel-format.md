# Novel Format Rules

正文和大纲文件的格式规范。当用户编辑正文或大纲文件时自动加载。

## 绝对禁止

1. **禁止段落超过 60 字**：一句一段，每段不超过 60 个汉字（对话和内心独白除外）。
2. **禁止段落间空行**：正文段落之间不加空行，保持紧密节奏。
3. **禁止「他说」「她道」**：对话用动作代替对话标签。
4. **禁止大段描写堆砌**：描写必须穿插动作或对话，不连续超过 3 段纯描写。
5. **禁止跳出视角**：已确定视角后，不切换到其他角色内心。

## 格式规范

- 对话独立成行，用半角双引号
- 段落以情绪或动作为单位划分
- 章节标题用 `## 第X章 章名` 格式
- 正文段落之间不加空行
- 章节之间不使用 `---` 或水平分隔线

## Examples

### Correct — 一句一段
```
沈栀抬手，灵力从指尖涌出。
面前的结界出现一道裂缝，碎纹像蛛网般蔓延开。
她咬紧牙关加大输出，整条手臂开始发颤。
```

### Wrong — 段落超长
```
沈栀抬手灵力从指尖涌出面前的结界出现一道裂缝碎纹像蛛网般蔓延开她咬紧牙关加大输出整条手臂开始发颤最后结界轰然碎裂碎片向四周飞溅。
```

### Correct — 对话用动作引出
```
沈栀将茶杯往桌上一顿。
"你到底想说什么？"
陆衍没接话，只是看着窗外。
```

### Wrong — 使用对话标签
```
她说道："你到底想说什么？"
他叹道："没什么。"
```

## 大纲格式规范

### 整体大纲（8 部分必填）

1. **核心设定**：一句话卖点 + 题材类型 + 目标平台
2. **角色设计**：主角 + 女主矩阵 + 反派体系
3. **主线剧情**：4-6 卷架构，每卷核心冲突
4. **感情线设计**：每条感情线的起点/发展/高潮/结局
5. **吸引元素**：亲密/幽默/惊喜/日常的密度规划
6. **节奏控制**：高潮/低谷交替、情绪曲线
7. **支线设计**：支线与主线的交织点
8. **字数分配**：各卷/各部分的字数规划

### 章节大纲（每章必含）

1. **章节号和标题**
2. **章节类型**：开篇/日常/亲密/高潮/转折
3. **情绪目标**：本章要让读者感受到什么
4. **核心事件**：推动情节的关键事件
5. **出场角色**：本章涉及的角色
6. **钩子/悬念**：章末钩子类型

### 章末钩子类型（至少用一种）

1. 悬念型：留下未解的问题
2. 反转型：出人意料的转折
3. 冲突型：新的矛盾出现
4. 情感型：情绪高点或低点
5. 揭秘型：揭露隐藏信息
6. 威胁型：危险逼近
7. 选择型：角色面临两难
