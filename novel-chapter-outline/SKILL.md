# 小说章节大纲迭代生成系统

**核心理念**：章节大纲不是一次生成的，而是通过"生成→评分→优化"的迭代循环打磨出来的。

**本skill整合全部创作工具**，自动完成全流程。

---

## 整合工具清单

| 工具 | 整合阶段 | 作用 |
|:---|:---|:---|
| **内部头脑风暴** | 阶段一：节奏构思 | 探索10章的节奏安排 |
| **WebSearch** | 阶段二：资料支撑 | 搜索专业细节、背景资料 |
| **novel-worldbuilding** | 阶段二+六 | 读取/更新世界观设定 |
| **novel-writing** | 阶段三：大纲生成 | 提供亲密/幽默/日常/悬念设计技法 |
| **novel-evaluation** | 阶段四：节奏评分 | 章节大纲评价体系 |
| **novel-optimization** | 阶段五：节奏优化 | 针对短板的优化技法 |

> **说明**：本工具已移除伪整合，改为内部实现或真实工具调用。

## 使用方式

### 基础用法
- `/novel-chapter-outline [大纲文件路径]` — 基于整体大纲生成下10章详细大纲
- `/novel-chapter-outline [大纲文件路径] --current [已写章节文件路径]` — 基于已写内容生成后续10章

### 进阶用法
- `/novel-chapter-outline [大纲文件路径] --from 第XX章` — 从指定章节开始生成
- `/novel-chapter-outline [大纲文件路径] --focus 亲密` — 这10章重点强化亲密描写
- `/novel-chapter-outline [大纲文件路径] --focus 幽默` — 这10章重点强化幽默元素
- `/novel-chapter-outline [大纲文件路径] --focus 惊喜` — 这10章重点强化意外反转

### 流程控制
- `/novel-chapter-outline --max 3` — 最多迭代3轮（默认5轮）
- `/novel-chapter-outline --target 8.5` — 目标分数8.5（默认9.0）

### 达标标准
- **默认目标分数：9.0**（极品章节大纲）
- **达标判定：总分 ≥ 9.0** 才能采用
- **未达标处理**：继续优化迭代，直到达标或达到最大轮数

## 阶段一：节奏构思（brainstorming）

**工具**：brainstorming（头脑风暴）

**目标**：在正式生成章节大纲前，先进行头脑风暴，探索10章的最佳节奏安排。

### 执行流程

```
1. 读取整体大纲，理解当前所在剧情弧
2. 根据剧情位置（开篇/发展/高潮/转折），进行发散思考
3. 生成3-5个不同的节奏安排方向，每个方向包含：
   - 10章的起承转合设计
   - 亲密节奏安排（小雨→中雨→大雨→雷鸣的分布）
   - 高潮点位置（在哪里设置高潮）
   - 悬念钩子分布（如何驱动追更）
4. 用户选择一个方向，或系统自动选择最有潜力的
```

### 输出格式

```markdown
# 章节节奏构思头脑风暴

## 章节范围：第XX章 - 第XX章

## 所属剧情弧：[整体大纲中的哪个弧]

## 核心任务：[这10章要完成什么]

## 节奏安排方向

### 方向A：[名称]
- 起承转合：[如何安排]
- 亲密节奏：[分布]
- 高潮点位置：[在哪里]
- 悬念钩子：[如何驱动]

### 方向B：[名称]
[同上]

### 方向C：[名称]
[同上]

## 推荐选择
[分析哪个方向最有潜力，为什么]
```


## 阶段三：大纲生成（novel-writing）

**工具**：novel-writing（创作技法工具箱）

**目标**：基于节奏构思和资料支撑，生成完整的10章详细大纲。

### 10章定位

10章 ≈ 2-3万字，是一个**中型叙事单元**：
- 不需要完成一个完整故事弧
- 但需要有**起承转合**的节奏
- 要有**承上启下**的功能
- 要有**至少1-2个高潮点**

### 10章节奏模板

| 章节 | 功能 | 吸引力元素 | 说明 |
|------|------|------------|------|
| 第1章 | 承上 | 悬念钩子 | 接续前文，抛出新钩子 |
| 第2章 | 铺垫 | 日常+幽默 | 建立氛围，展示角色魅力 |
| 第3章 | 升温 | 亲密升温 | 小雨级别，制造暧昧 |
| 第4章 | 推进 | 悬念推进 | 信息揭露，推动剧情 |
| 第5章 | 互动 | 亲密+幽默 | 中雨级别+笑点 |
| 第6章 | 冲突 | 紧张感 | 制造矛盾/危机 |
| 第7章 | 减压 | 日常+幽默 | 悬念后的放松，减压阀 |
| 第8章 | 高潮 | 亲密/情感高潮 | 大雨级别/情感爆发 |
| 第9章 | 转折 | 惊喜反转 | 出人意料的发展 |
| 第10章 | 启下 | 悬念钩子 | 章末钩子，驱动追更 |

### 吸引力元素密度（10章内）

| 元素 | 最低数量 | 目标数量 | 说明 |
|------|----------|----------|------|
| 亲密场景 | 2个 | 3-4个 | 必须有渐进升级 |
| 笑点 | 3个 | 5-8个 | 多种类型交替 |
| 日常场景 | 2个 | 3-4个 | 烟火气+角色魅力 |
| 悬念钩子 | 5个 | 10个 | 每章结尾都要有 |
| 惊喜反转 | 0个 | 1-2个 | 可选，看剧情位置 |

### 大纲框架

#### 第一部分：10章总览
- 章节范围、所属剧情弧、核心任务
- 承接上文、启下铺垫
- 感情线进展规划
- 吸引力元素规划
- 字数预估

#### 第二部分：逐章详细大纲（10章）
每章包含：
- 章节功能（承上/铺垫/升温/推进/互动/冲突/减压/高潮/转折/启下）
- 场景设定（时间、地点、在场人物）
- 剧情内容（200-300字）
- 吸引力元素（亲密/幽默/日常/悬念/惊喜）
- 承上启下设计
- 预计字数

#### 第三部分：重点场景设计（2-3个）
针对关键场景提供详细设计：
- 场景目的
- 详细设计（氛围、感官、角色反应、情感层次、对话、动作、内心）
- 技法运用
- 预期读者反应

## 阶段四：节奏评分（novel-evaluation）

**工具**：novel-evaluation（评价体系）

**目标**：量化评估章节大纲的节奏设计，识别短板。

### 评分维度

| 维度 | 权重 | 核心问题 | 评分标准 |
|------|------|----------|----------|
| **节奏完整性** | 25% | 10章是否有完整的起承转合？ | 9-10:完美起承转合 7-8:基本完整 5-6:一般 3-4:不完整 1-2:无节奏 |
| **亲密节奏** | 25% | 亲密场景分布是否合理？有渐进升级？ | 9-10:渐进升级 7-8:有阶段 5-6:一般 3-4:生硬 1-2:无 |
| **悬念密度** | 20% | 每章是否有钩子？能否驱动追更？ | 9-10:每章都有 7-8:大部分有 5-6:偏少 3-4:稀疏 1-2:无 |
| **幽默分布** | 15% | 笑点是否均匀分布？有"减压阀"？ | 9-10:均匀分布 7-8:合理 5-6:偏少 3-4:生硬 1-2:无 |
| **承上启下** | 10% | 是否能接续前文并为后续铺垫？ | 9-10:完美衔接 7-8:基本连贯 5-6:一般 3-4:有断裂 1-2:脱节 |
| **场景详细度** | 5% | 重点场景是否有足够细节？ | 9-10:非常详细 7-8:详细 5-6:一般 3-4:粗略 1-2:无细节 |

### 评分计算

```
总分 = 节奏完整性×25% + 亲密节奏×25% + 悬念密度×20% + 幽默分布×15% + 承上启下×10% + 场景详细度×5%
```

### 评分标准

| 总分 | 等级 | 判定 | 后续动作 |
|------|------|------|----------|
| ≥ 9.0 | 极品 | **✓ 采用** | 进入阶段六深度分析，输出最终文件 |
| 8.0-8.9 | 优质 | 需要继续优化 | 进入阶段五优化循环 |
| 7.0-7.9 | 良好 | 需要大幅优化 | 进入阶段五优化循环 |
| 6.0-6.9 | 一般 | 需要大幅优化 | 进入阶段五优化循环 |
| < 6.0 | 较差 | 重新生成 | 可选择重生成或优化 |

**重要：只有 ≥ 9.0 分的章节大纲才会被采用并保存为最终文件。**

## 阶段五：节奏优化（novel-optimization）

**工具**：novel-optimization（优化策略）

**目标**：针对评分短板进行定向优化，补充新的钩子元素。

### 优化策略（根据短板选择）

#### 节奏完整性优化（权重25%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 缺乏起承转合 | 叙事节奏控制 | 模块C | 设计完整的10章节奏曲线 |
| 缺乏高潮点 | 高潮场景设计 | 模块B | 在第8章设置高潮 |
| 缺乏减压阀 | 幽默技巧 | 模块C | 在紧张后插入轻松场景 |

#### 亲密节奏优化（权重25%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 分布不均 | 渐进式隐喻系统 | 模块C | 设计小雨→中雨→大雨→雷鸣的递进 |
| 缺乏升级感 | 渐进式推进法 | 模块B | 设计四阶段亲密递进 |
| 缺乏禁忌感 | 身份禁忌极致化 | 模块C | 设计"绝对不能碰"的禁忌线 |
| 缺乏克制 | 点到为止的克制 | 模块C | 设计"差一点"的克制场景 |

#### 悬念密度优化（权重20%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 钩子少 | 钩子设置技巧 | 模块B | 每章结尾设计钩子 |
| 类型单一 | 信息差驱动法 | 模块B | 设计多种钩子类型交替 |
| 缺乏驱动力 | 嵌套式叙事框架 | 模块B | 设计贯穿10章的悬念线 |

#### 幽默分布优化（权重15%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 笑点少 | 幽默技巧 | 模块C | 添加内心独白式幽默 |
| 分布不均 | 破折号旁白法 | 模块C | 在高压段后插入减压阀 |
| 类型单一 | 对比式幽默 | 模块C | 设计多种幽默类型交替 |

#### 承上启下优化（权重10%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 承接断裂 | 场景延续法 | 模块C | 第1章接续前文场景 |
| 铺垫不足 | 钩子设置技巧 | 模块B | 第10章为后续铺垫 |
| 连续性差 | 信息差驱动法 | 模块B | 保持线索的连续性 |

#### 场景详细度优化（权重5%）

| 短板类型 | 优化技法 | 来源 | 优化方法 |
|----------|----------|------|----------|
| 细节不足 | 感官叠加法 | 模块C | 增加多感官描写 |
| 缺乏氛围 | 环境烘托法 | 模块C | 增加环境描写 |
| 反应单一 | 微反应刻画法 | 模块C | 增加角色微反应 |

## 迭代控制逻辑

### 迭代判定规则

```python
# 伪代码逻辑
target_score = 9.0  # 默认目标分数（必须达到9分才能采用）
max_rounds = 5      # 默认最大轮数
current_round = 0
best_score = 0
best_outline = None

# 读取整体大纲和世界观设定
overall_outline = read_outline_file(大纲文件路径)
worldbuilding = read_worldbuilding()

# 阶段一：节奏构思（内部发散思考）
selected_direction = user_select_or_auto_best(节奏安排方向列表)

# 阶段二：资料支撑
research_data = web_search(需要查询的资料)

while current_round < max_rounds:
    current_round += 1

    if current_round == 1:
        # 阶段三：初始生成
        chapter_outline = novel_writing_generate(overall_outline, selected_direction, research_data)
    else:
        # 阶段五：定向优化
        chapter_outline = optimize_chapter_outline(chapter_outline, weaknesses)

    # 阶段四：评分
    score_report = novel_evaluation_score(chapter_outline)
    current_score = score_report.total_score

    # 记录最优版本
    if current_score > best_score:
        best_score = current_score
        best_outline = chapter_outline

    # 判断是否达标
    if current_score >= target_score:
        # 阶段六：深度分析 + 世界观更新
        deep_analysis = analyze_chapter_outline(chapter_outline)
        update_worldbuilding_foreshadowing(chapter_outline)
        output_final_outline(chapter_outline, score_report, deep_analysis, versions)
        break

    # 输出当轮报告
    output_round_report(chapter_outline, score_report, current_round)

# 达到最大轮数，输出最优版本（标注未达标）
if current_round == max_rounds and best_score < target_score:
    deep_analysis = analyze_chapter_outline(best_outline)
    update_worldbuilding_foreshadowing(best_outline)
    output_best_outline(best_outline, best_score, versions, note="未达标")
```

## 输出文档规则

### 重要：只保留最终达标文件

**本skill只保存最终达标的章节大纲文件（9.0+分），中间迭代过程不保存为文件。**

### 输出路径
- 默认目录：`~/小说/`（跨平台兼容，可通过项目 CLAUDE.md 或参数自定义）
- 文件命名：`chapter_outline_[章节范围]_[时间戳].md`

### 保存条件
- **只有总分 ≥ 9.0 的章节大纲才会保存为文件**
- 未达标的迭代过程仅在对话中显示，不保存文件
- 达到最大轮数仍未达标时，保存当前最优版本（需标注"未达标"）

### 输出内容
最终文件包含：
1. 10章总览
2. 逐章详细大纲（10章）
3. 重点场景设计（2-3个）
4. 本批次总结
5. 最终评分报告
6. 迭代历程总结

### 输出确认
保存文件后，向用户报告：
- 文件保存路径
- 章节范围（第XX章 - 第XX章）
- 迭代轮次和最终分数
- 核心任务概述
- 吸引力元素统计（亲密场景X个、笑点X个、悬念钩子X个）
- 预计字数

## 大纲生成规则

### 1. 承上启下原则
- **承上**：第1章必须接续前文，不能有断裂感
- **启下**：第10章必须为后续铺垫，不能有完结感
- **连续性**：角色状态、情感进展、剧情线索必须连贯

### 2. 吸引力元素密度
- **亲密场景**：至少2个，目标3-4个，必须有渐进升级
- **笑点**：至少3个，目标5-8个，多种类型交替
- **日常场景**：至少2个，目标3-4个，有烟火气
- **悬念钩子**：每章结尾都要有
- **惊喜反转**：可选，1-2个

### 3. 节奏控制
- **张弛有度**：高压段后必有低压段
- **减压阀**：紧张/冲突后必须有轻松/幽默
- **渐进升级**：亲密/冲突都要逐步升级

### 4. 技法运用
- 亲密场景：参照novel-writing模块C的亲密场景写作技法
- 幽默场景：参照novel-writing模块C的幽默技巧
- 日常场景：参照novel-writing模块C的场景构建
- 悬念钩子：参照novel-writing模块B的钩子设置技巧
