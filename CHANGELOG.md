## [9.7.0] - 2026-06-02

### 新增：场景设计系统

#### 问题
- 使用skill创作的小说每章只有一个场景
- 没有场景切换，像流水账
- 场景之间缺乏过渡，读者体验差

#### 解决方案

**1. 场景设计原则**
- 每章必须包含多个场景（2-6个）
- 场景 = 同一时间 + 同一地点 + 同一组人物
- 场景类型：对话场景、动作场景、描写场景、回忆场景、过渡场景、转折场景

**2. 场景切换技巧**
- 硬切（直接切换）
- 情绪衔接（情绪延续过渡）
- 意象过渡（意象连接）
- 时间跳跃（时间标记切换）
- 空间并置（蒙太奇效果）

**3. 场景数量建议**
- 日常章节：2-3个场景
- 冲突章节：3-5个场景
- 高潮章节：4-6个场景
- 过渡章节：1-2个场景

**4. 追踪系统更新**
- 情节追踪：记录每章场景列表和切换方式
- 角色追踪：记录角色在各场景的状态变化
- 章节大纲：每章需包含场景分解

## [9.5.0] - 2026-06-02

### 新增：多Agent并行创作

#### 问题
- 全流程创作只在一个会话窗口中进行
- 创作速度慢，效率低

#### 解决方案

**正文阶段支持多Agent并行创作**

**1. 并行条件**
- 章节大纲已完成（知道每章写什么）
- 角色系统和世界线系统已建立
- 追踪文件已初始化

**2. 并行策略**

按故事弧并行：
```
故事弧（第1-15章）
    ↓
Agent 1：第1-5章
Agent 2：第6-10章
Agent 3：第11-15章
    ↓
合并结果 → 统一检查 → 更新追踪文件
```

按独立剧情线并行：
```
主线剧情（Agent 1）
支线A（Agent 2）
支线B（Agent 3）
    ↓
合并结果 → 交叉检查 → 更新追踪文件
```

**3. 并行Agent工作流程**
- 读取统一的参考文件（角色系统、世界线系统、章节大纲、追踪文件）
- 按照统一的规则创作（技法库、去耦合组件、去AI味规则）
- 输出到指定位置（不更新追踪文件）

**4. 并行后的合并流程**
- 顺序检查：按章节顺序检查衔接点
- 一致性检查：角色名、时间线、伏笔是否一致
- 追踪文件更新：统一更新情节追踪、角色追踪、上下文摘要
- 批次优化：对整个故事弧进行对比优化

**5. 效率提升**

| 原来 | 并行后 |
|------|--------|
| 15章串行：约15个时间单位 | 3个Agent并行：约5个时间单位 |
| 30章串行：约30个时间单位 | 5个Agent并行：约6个时间单位 |

#### 修改的文件
- `SKILL.md` — 全流程章节新增"并行创作机制"

## [9.4.0] - 2026-06-02

### 修复：短句过多问题

#### 问题
- 每一章的短句太多，一句话就是一段
- 缺乏长短句结合，节奏单一
- 例如："她想。她居然获得了提名。英国电影学院奖。最佳编剧。"

#### 解决方案

**1. 更新节奏规则**

核心原则：长短句交错，节奏有起伏。

| 场景 | 句长 | 示例 |
|------|------|------|
| 打斗/紧张 | 3-8 字 | 他侧身。刀光一闪。血溅在墙上。 |
| 对话 | 口语化 | "你疯了？""可能吧。" |
| 日常 | 8-15 字 | 厨房里有炒菜的声音，闻着像辣椒炒肉。 |
| 描写 | 15-25 字 | 天快黑了，巷子里没什么人，只有远处传来几声狗叫。 |
| 情感/心理 | 20-40 字 | 她站在窗前，看着外面的雨，想起那年夏天他撑着伞走过来的样子。 |

**2. 段落长度控制**

- 一个段落通常包含2-4个句子
- 不要一句话一段（太碎）
- 不要超过6句话一段（太长）
- 根据内容自然分段，不要机械断句

**3. 节奏控制技巧**

- 短句制造紧张：连续短句用于打斗、冲突、紧张场景
- 长句营造氛围：长句用于描写、情感、回忆场景
- 长短交错：不要连续5句以上都是短句或长句
- 关键信息用短句：重要转折、情感冲击用短句强调
- 铺垫用长句：背景介绍、环境描写用长句展开

**4. 更新去AI味规则**

Gate D 新增检测：
- 连续5句以上都是短句（<10字）→ 合并或添加长句
- 连续5句以上都是长句（>20字）→ 拆分或添加短句
- 一句话一段 → 合并到相邻段落

#### 修改的文件
- `rules/anti-ai-writing.md` — 更新规则3为"长短句结合"，添加示例和技巧
- `rules/deslop-rules.md` — Gate D 新增短句/长句检测

## [9.3.0] - 2026-06-02

### 修复：故事弧公式化问题

#### 问题
- 故事弧总是在整数章结束（第100章、第300章）
- 固定10章一批的结构导致公式化
- 读者能感觉到"为了凑整数而拖沓或赶工"

#### 解决方案

**1. 反对公式化**

| 公式化做法 | 问题 | 正确做法 |
|------------|------|----------|
| 第100章第一卷结束 | 太刻意 | 在剧情自然转折点结束（如第87章、第113章） |
| 第300章全书完结 | 为了凑整数 | 故事讲完就结束（如第267章、第342章） |
| 每卷固定章数 | 削足适履 | 根据剧情需要决定每卷章数 |

**2. 故事弧结束标准**

故事弧应该在自然叙事节点结束：
- 冲突解决型：主要反派被击败、核心矛盾解决
- 关系转变型：主角关系发生重大变化
- 世界观揭示型：重要秘密被揭开
- 成长蜕变型：主角完成阶段性成长
- 情感高潮型：重大情感事件

**3. 灵活的批次处理**

不再固定10章一批，根据故事弧灵活调整：

| 故事弧类型 | 建议章数 | 说明 |
|------------|----------|------|
| 短弧（小冲突） | 3-7章 | 快节奏，密集冲突 |
| 中弧（阶段性冲突） | 8-15章 | 标准节奏，完整发展 |
| 长弧（重大冲突） | 16-30章 | 慢节奏，深度展开 |
| 超长弧（世界观级） | 30-50章 | 大型故事线 |

#### 修改的文件
- `SKILL.md` — 正文生成章节新增"故事弧设计原则"，更新批次处理逻辑

## [9.2.0] - 2026-06-02

### 新增：章节标题设计系统

#### 问题
章节标题过于平淡、无信息量、缺乏钩子效果。

#### 解决方案

**六种高质量标题类型**：

1. **情感独白型**（角色心声）
   - 示例："朕这一生如履薄冰"、"我什么都不剩下了"
   - 效果：建立情感连接

2. **悬念提问型**（制造好奇心）
   - 示例："如果她知道了一切，会有什么反应？"
   - 效果：让读者带着问题进入章节

3. **数字引爆型**（具体化冲击）
   - 示例："总分——555分！"、"——7061.83"
   - 效果：用具体数字制造戏剧性

4. **反差对比型**（制造冲突）
   - 示例："婚礼当天，新娘逃了"
   - 效果：用对比制造张力

5. **意象隐喻型**（氛围营造）
   - 示例："雨夜的告别"、"最后一片落叶"
   - 效果：用意象营造氛围

6. **金句标志型**（记忆点）
   - 示例："这一刀，我记了三年"
   - 效果：创造标志性记忆点

**标题创作流程**：
1. 读完本章内容 → 提取核心情感/冲突/悬念
2. 选择标题类型 → 根据本章特点选择最合适的类型
3. 创作3-5个备选 → 从不同角度创作
4. 选择最有钩子的 → 让读者看目录就想翻页
5. 检查与前后章的连贯 → 标题之间形成节奏

**标题禁忌**：
- ❌ 过于直白："林风眠遇到柳媚"
- ❌ 过于抽象："命运的齿轮"
- ❌ 无信息量："第一章"
- ❌ 剧透结局："林风眠战胜柳媚"
- ❌ 与内容不符

#### 修改的文件
- `SKILL.md` — 正文生成章节新增"章节标题设计"

## [9.1.0] - 2026-06-02

### 增强：去耦合化组合机制

#### 新增功能

**1. 多维度组合**（不止基础维度）
- 角色成长阶段（新手/成长/成熟/巅峰/衰退）
- 关系变化节点（初遇/试探/冲突/和解/分离/重逢）
- 信息不对称（谁知道什么、谁知道谁知道什么）
- 时间压力（紧迫/舒缓/停滞/跳跃）
- 空间特殊性（封闭/开放/熟悉/陌生/象征性空间）

**2. 学习优秀小说的设计**
- 组合时参考优秀小说的处理方式
- 学习其"组合逻辑"而非直接模仿
- 用不同的组件重新组合

**3. 创造惊喜的组合原则**
- 反差组合：看似矛盾的组件
- 跨场景组合：将A场景的组件用到B场景
- 渐进组合：同一角色逐步展现不同面
- 对比组合：同一场景不同角色的反差
- 意外组合：打破读者预期

**4. 多样性检查**
- 与前几次组合是否有差异
- 是否使用了新的组件
- 是否打破了读者的预期
- 是否创造了新的角色面/情节可能性

#### 修改的文件
- `SKILL.md` — 去耦合化组合章节新增组合增强机制
